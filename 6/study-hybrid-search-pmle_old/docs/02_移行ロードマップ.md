# 02_移行ロードマップ

本ドキュメントは Phase 6 (`study-hybrid-search-pmle`) の決定的仕様を管理する。Phase 6 は **Phase 7 を最新完成形の基準としたうえで、serving 差分を除いた Phase 5 系の構成に、採用を継続する PMLE 追加技術だけを載せる** フェーズとする。

Phase 6 の目的は、Phase 5 の不動産ハイブリッド検索コードを壊さずに、PMLE 学習に有効な技術を **実コードへ bolt-on で統合して学ぶ** こと。削除済みの旧追加機能は本フェーズの採用対象外とし、関連コード・Terraform・文書は削除する。

## 0. 不変ルール

以下は User 合意なしに変更しない。

- 題材は不動産ハイブリッド検索のまま維持する
- `/search` の既定挙動は Phase 5 と同じに保つ
- Phase 3-7 共通の中核構成 `LightGBM + multilingual-e5 + Meilisearch + BigQuery VECTOR_SEARCH + RRF` を維持する
- Phase 6 の追加技術は opt-in に限定し、既定経路を置換しない
- IaC は Terraform を正とし、手動作成を前提にしない

## 1. スコープ

### 1.1 採用

Phase 6 で扱う追加技術は次の 6 つ。

| # | 技術 | 方針 |
|---|---|---|
| T1 | BQML | `/search` の補助スコアとして扱う |
| T2 | Dataflow | ranking log の並列集計経路として追加する |
| T4 | Explainable AI | reranker explain を opt-in で追加する |
| T5 | Monitoring SLO | Cloud Run `search-api` の運用監視に追加する |
| T6 | RAG (Gemini) | `/rag` を新設し `/search` とは分離する |
| T8 | Model Garden (Gemini enrichment) | 物件説明文の補助構造化として別テーブルへ出力する |

### 1.2 不採用

以下は Phase 6 の対象外とし、残骸があれば削除する。

- 削除済みの旧追加機能
- Phase 7 の `GKE` / `KServe` / `Gateway API`
- W&B / Looker Studio / Doppler を教材主対象として復活させる変更

## 2. 実装方針

- Phase 7 を最新完成系の基準とし、Phase 6 では serving 層を Cloud Run + Vertex Endpoint 系に固定する
- Phase 6 の追加技術は既存経路に対する `bolt-on` として実装する
- 既定値では常に Phase 5 の `/search` 挙動を返す
- 新技術は新 endpoint、補助フィールド、別テーブル、監視、別 pipeline として追加する
- `Meilisearch` は学習用ローカル/小規模構成で扱いやすく、`Elasticsearch` より導入しやすいため Phase 3-7 を通じて採用維持とする

### 2.1 検証方針

- 移行中は差分を意識しつつ、コード修正の速度を優先する
- E2E や CI/CD 上の総合検証は後段へ回す
- ローカルで回せる単体テスト、限定的な lint、局所的な確認を優先する
- 既存の lint / test 負債は、今回の移行差分と無関係なら別件として切り分ける
- まずは Phase 6 を現行スコープへ戻すことを優先し、完全検証は後続フェーズでまとめて行う

## 3. 統合トピック

| # | 技術 | 統合先 | 既定挙動への影響 |
|---|---|---|---|
| T1 | BQML | `PopularityScorer` adapter として `/search` レスポンスへ補助スコア追加 | なし |
| T2 | Dataflow | `ml/streaming/` に ranking-log 集計パイプライン追加 | なし |
| T4 | Explainable AI | reranker explain と `/search?explain=true` を追加 | なし |
| T5 | Monitoring SLO | `infra/terraform/modules/slo/` と `ops-slo-status` を追加 | なし |
| T6 | RAG (Gemini) | `/rag` endpoint と summarizer を追加 | なし |
| T8 | Model Garden (Gemini enrichment) | `properties_enriched` 生成処理を追加 | なし |

### 3.1 既定値

Phase 6 の既定値は Phase 5 互換を守る。

```env
ENABLE_RAG=false
BQML_POPULARITY_ENABLED=false
explain=false
enable_streaming=false
enable_streaming_job=false
```

## 4. Phase 6 修正方針

### 4.1 コード

- 削除済みの旧追加機能に紐づく adapter、切替フラグ、Terraform module、設定値を削除する
- `/search` の composition root は `Meilisearch + BigQuerySemanticSearch + LightGBM rerank` を既定かつ唯一の中核経路として維持する

### 4.2 Terraform

- 削除済み機能に紐づく Terraform module を削除する
- `environments/dev` から関連 module / variable / output を削除する
- `slo` と `streaming` は維持する

### 4.3 文書

- `README.md` / `CLAUDE.md` / `docs/01_仕様と設計.md` から旧スコープ前提の記述を除去する
- Phase 6 の対象技術は 6 テーマに統一する
- 教育資料では削除済み機能を学習対象として扱わない
