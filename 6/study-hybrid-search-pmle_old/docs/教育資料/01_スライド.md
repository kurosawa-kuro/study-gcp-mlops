# PMLE 技術統合編

Phase 5 を土台に  
**追加技術を実コードへ統合して学ぶ**

---

## この Phase の主眼

- Phase 5 の完成形を壊さずに、新技術を追加する
- 追加技術を `adapter` / `副経路` / `追加エンドポイント` として統合する
- `/search` の default 挙動は Phase 5 のまま維持する

---

## 不変の中核

この Phase でも変えないもの:

- `Meilisearch`
- `BigQuery VECTOR_SEARCH`
- `multilingual-e5`
- `RRF`
- `LightGBM LambdaRank`

> 新技術は学習用に足すが、検索の中核は置き換えない。

---

## 追加して学ぶ技術

| 技術 | 学ぶ観点 |
|---|---|
| `BQML` | BigQuery 上の ML 活用 |
| `Dataflow` | ストリーミング処理 |
| `Monitoring SLO` | 運用指標と burn-rate |
| `Gemini RAG` | 生成系の副経路 |
| Explainable AI | 予測根拠の見せ方 |
| Model Garden | 補助的な生成処理 |

---

## 教材で対象外にしたもの

- `W&B`
- `Doppler`

削除済み機能やこれらの周辺ツールは教育資料の主対象にしない。  
古い記述が残っていれば削除する。

---

## どう統合するか

```text
既存の /search
  ├─ default: Phase 5 のまま
  └─ opt-in: 追加技術を有効化
```

例:

- explain 付き検索
- RAG エンドポイント
- popularity 補助スコア

---

## Port/Adapter のまま拡張する

- 新 Port を足す
- 既存 Port に alternative adapter を足す
- feature flag で切り替える

これにより、  
**追加学習と既存挙動維持を両立** する。

---

## 学習者が見るべきポイント

1. 中核を変えずに技術を足す考え方
2. 本流と副経路を分ける考え方
3. 追加機能を opt-in にする理由
4. 監視や生成系を検索本体と切り離す設計

---

## この Phase で押さえること

1. PMLE 学習は「別プロジェクト化」ではなく既存コードへの統合で学ぶ
2. default 挙動は Phase 5 を維持する
3. 新技術は Port/Adapter の延長で扱う

---
