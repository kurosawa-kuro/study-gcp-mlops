### T1. BQML による補助スコア

**目的**: `CREATE MODEL` + `ML.PREDICT` を実コードで触る。Phase 5 rerank の並び順には手を入れず、**レスポンスの補助フィールド** として popularity を surface して parity cascade を回避。

| 新 Adapter | `app/services/adapters/bqml_popularity_scorer.py::BQMLPopularityScorer` (`ML.PREDICT`) |
| BQML モデル | `scripts/bqml/train_popularity.sql` (`mlops.property_popularity`、`BOOSTED_TREE_REGRESSOR`、`input_label_cols=['ctr']`、`enable_global_explain=TRUE`) |
| 再学習 CLI | `scripts/local/bqml/train_popularity.py` (`bq query` 実行) |

### T2. Dataflow streaming (ranking-log → hourly CTR)

**目的**: Apache Beam Python で Pub/Sub → 1h tumbling window 集計 → BQ の一連を Flex Template で運用する経験。既存の BQ Subscription (raw ranking_log) は維持し、**並列経路**として 1 時間 CTR テーブルを追加。

| 要素 | ファイル / 値 |
|---|---|
| Beam pipeline | `ml/streaming/pipeline.py` (lazy import、ruff / mypy は Beam 非依存で通る) |
| Flex Template | `ml/streaming/container/Dockerfile` + `ml/streaming/container/requirements.txt` + `ml/streaming/container/flex_template_metadata.json` |
| 新 BQ テーブル | `mlops.ranking_log_hourly_ctr` (`infra/terraform/modules/data/main.tf`、DAY partition on `window_start`) |
| 新 Terraform モジュール | `infra/terraform/modules/streaming/` (sa-dataflow + IAM + optional `google_dataflow_flex_template_job` on google-beta) |
| Terraform 変数 | `enable_streaming: bool = false` / `enable_streaming_job: bool = false` / `streaming_flex_template_gcs_path: string` |
| SA | `sa-dataflow` (pubsub.subscriber + dataflow.worker + storage.objectAdmin + bigquery.dataEditor + bigquery.jobUser) |
| pipeline パラメータ | `input_topic=projects/.../topics/ranking-log` / `output_table=mlops-dev-a:mlops.ranking_log_hourly_ctr` / `window_size_sec=3600` |

**中核への影響**: なし (別テーブル・別 SA・Pub/Sub は subscribe only)。

### T3. Vertex Vector Search (alternative semantic backend)

**目的**: Matching Engine (ANN) を BQ `VECTOR_SEARCH` の alternative として adapter 追加。`SemanticSearchPort` を正式に抽出し、Phase 5 の BQ 実装も同 Port 配下の adapter として整理。

| 要素 | ファイル / 値 |
|---|---|
| 新 Port | `app/services/protocols/semantic_search.py::SemanticSearchPort` |
| adapter #1 (Phase 5 既存挙動) | `app/services/adapters/semantic_search.py::BigQuerySemanticSearch` |
| adapter #2 (新規) | `app/services/adapters/semantic_search.py::VertexVectorSearchSemantic` (Matching Engine `find_neighbors` + BQ post-filter) |
| `BigQueryCandidateRetriever` | `semantic: SemanticSearchPort \| None` 引数を受け取るように変更。後方互換 default = `BigQuerySemanticSearch` |
| 新 Terraform モジュール | `infra/terraform/modules/vector_search/` (`google_vertex_ai_index` + `google_vertex_ai_index_endpoint`、`approximate_neighbors_count=150`、Tree-AH、`COSINE_DISTANCE`) |
| Terraform 変数 | `enable_vector_search: bool = false` / `vector_search_embedding_dimensions=768` / `vector_search_contents_delta_uri` |
| config flag | `ApiSettings.semantic_backend: Literal["bq","vertex"] = "bq"` / `vertex_vector_search_index_endpoint_id` / `vertex_vector_search_deployed_index_id` |
| composition root | `app/main.py::_build_semantic_search` (flag=vertex かつ endpoint/deployed 両方設定でのみ切替、不足時は BQ fallback + 警告 log) |

**中核への影響**: なし (default bq、response schema 互換)。

### T4. Explainable AI (reranker attribution)

**目的**: LightGBM の TreeSHAP (`pred_contrib=True`) で per-instance feature attribution を計算。Vertex の `ExplanationMetadata` を組まずに `/predict` に `parameters.explain=true` で要求する省略形と、純正 `/explain` route の両方を用意。

| 要素 | ファイル / 値 |
|---|---|
| reranker server 変更 | `ml/serving/reranker.py` — `RerankerParameters{explain, feature_names}`、`RerankerResponse.attributions: list[dict] \| None`、新 `/explain` POST (`ExplainRequest` / `ExplainResponse`) |
| 新 Port | `RerankerExplainer` (既存 `RerankerClient` とは別の独立 Protocol) |
| adapter 拡張 | `VertexEndpointReranker.predict_with_explain(instances, feature_names) -> (scores, attributions)` (`Endpoint.predict(parameters={...})` 経由) |
| response 拡張 | `SearchResultItem.attributions: dict[str, float] \| None`、`_baseline` キーで LightGBM 期待値 offset |
| クエリ param | `/search?explain=true` (cache バイパス、`run_search(want_explanations=True)`) |
| ranking service | `RankedCandidate.attributions` を追加、`_score_with_explain` で adapter の Explainer 実装を優先 |
| ユニットテスト | `tests/unit/app/test_phase6_explain_and_rag.py` (explain あり/なし、Explainer 非対応 reranker の fall-back) |

**中核への影響**: なし (default `explain=false`、既存 `/search` レスポンス完全維持)。

### T5. Cloud Monitoring SLO + AlertPolicy

**目的**: Cloud Run の request_count / request_latencies メトリックに対して **フォーマルな SLO** を宣言、fast/slow burn-rate alert を配線する。Phase 5 の log-based alert (`api_error_rate` / `api_p95_latency`) と併存。

| 要素 | ファイル / 値 |
|---|---|
| 新 Terraform モジュール | `infra/terraform/modules/slo/` (`google_monitoring_custom_service` + `google_monitoring_slo` × 2 + `google_monitoring_alert_policy` × 4) |
| SLO #1 availability | `request_count` good_total_ratio、2xx/total、default goal 0.99 / 28d rolling |
| SLO #2 latency | `request_latencies` distribution_cut、default threshold 500ms / goal 0.95 / 28d rolling |
| alert policies | fast burn (`select_slo_burn_rate(..., 3600s) > 14.4`) × 2 + slow burn (`..., 259200s > 1.0`) × 2 |
| notification channel | `module.monitoring.notification_channel_id` を共有 (重複作成しない) |
| Terraform 変数 | `slo_availability_goal` / `slo_latency_threshold_ms` / `slo_latency_goal` / `slo_rolling_period_days` (28 default) |
| make target | `make ops-slo-status` (`scripts/local/ops/slo_status.py`、Monitoring REST API で compliance + 1h/3d burn-rate を取得) |
| outputs | `slo_service_id` / `slo_availability_name` / `slo_latency_name` |

**中核への影響**: なし (観測のみ、app コード無変更)。

### T7. Agent Builder 副経路 lexical

**目的**: Discovery Engine (Agent Builder の Search App) を **副経路 lexical retriever** として追加。Meilisearch は主経路のまま維持 (親リポ non-negotiable)。

| 要素 | ファイル / 値 |

| Terraform 変数 | `enable_agent_builder: bool = false` / `agent_builder_location="global"` / display name 2 種 |
| config flag | `lexical_backend: Literal["meili","agent_builder"] = "meili"` + `vertex_agent_builder_engine_id` + collection / serving_config ids |

## 4. Port / Adapter 境界と `make check` の扱い

Phase 6 で追加した要素は Phase 5 と同じ境界ルールに従う (実装済み):

- **Port**: `app/services/protocols/<name>.py` の Protocol。`protocols/__init__.py` で re-export (Phase 6 で `generator` / `popularity_scorer` / `semantic_search` / `RerankerExplainer` を追加)
- **Adapter**: `app/services/adapters/<name>.py`。外部 SDK (`google.cloud.*` / `vertexai.generative_models` / `google.cloud.discoveryengine_v1` / `apache_beam`) は adapter / `ml/streaming/` 内でのみ import
- **Service**: `app/services/<name>.py`。Port だけ import、adapter は import しない (`rag_summarizer` は `Generator` Port のみ)
- **境界検知**: `scripts/checks/layers.py::RULES` + `tests/unit/arch/test_import_boundaries.py` で CI 検知
- **mypy strict**: Phase 6 追加コード 103 source files すべて PASS

**現状の `make check`**: 255 passed / 1 failed (pre-existing parity path drift、Phase 6 とは無関係)。`make tf-validate` PASS (google + google-beta)。
