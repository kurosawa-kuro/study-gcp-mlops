## 0. 位置付け (引き算戦略上の Phase 4)

Phase 4 は親 [`README.md` §4](../../../README.md) の引き算チェーンで **Phase 5 から Vertex AI 標準 MLOps (Pipelines / Endpoint / Model Registry / Monitoring / Feature Store / Vector Search) を引き算した結果**。題材は Phase 3 から継承して不動産ハイブリッド検索を維持しつつ、**実行基盤を Local Docker Compose → GCP serverless** に移す phase。

中核は **GCP マネージドサービスの基礎習得** (Cloud Run / BigQuery / Pub/Sub / Secret Manager / Terraform / WIF) と、**BigQuery feature table / view + BigQuery `VECTOR_SEARCH`** によるサーバーレス検索構成。

---

## 1. Phase 3 → Phase 4 の差分

| 観点 | Phase 3 (Local) | Phase 4 (GCP serverless、追加) |
|---|---|---|
| 実行基盤 | Docker Compose | **Cloud Run Service / Jobs** |
| データ層 | PostgreSQL | **BigQuery** (検索 + 特徴量 + ログの中心) |
| ベクトル検索 | pgvector / 簡易 ANN | **BigQuery `VECTOR_SEARCH`** (Phase 5 で Vertex Vector Search に置換) |
| 特徴量管理 | local files | **BigQuery feature table / view** (Dataform) — Phase 5 Feature Store の入力源 |
| モデル成果物 | local filesystem | **GCS** |
| ログ送出 | PostgreSQL 直接 | **Pub/Sub → BigQuery** (BQ Subscription) |
| ranking 再学習 trigger | 手動 | **Cloud Scheduler + Eventarc + Cloud Function** (軽量 orchestration、Phase 5 まで継続。Phase 6 で Composer 本線昇格) |
| シークレット | local `.env` | **Secret Manager → Cloud Run secret injection (必須習得)** |
| IaC | なし | **Terraform** |
| CI/CD 認証 | なし | **WIF (Workload Identity Federation、SA Key 禁止)** |
| 監視 | 手動 | **Cloud Logging + Cloud Monitoring** |
| デプロイ | docker compose up | **GitHub Actions + Artifact Registry + Cloud Build** |

中核 5 要素 (Meilisearch BM25 / multilingual-e5 / ベクトルストア / RRF / LightGBM LambdaRank) は Phase 3 から継承して不変。**変えるのは実行基盤・認証・運用の責務分担のみ**。

## 2. Phase 4 の中核アーキテクチャ

```
[Cloud Scheduler]
   │ daily
   ▼
[Eventarc] ─→ [Cloud Function: pipeline-trigger]
                │
                ▼
[Cloud Run Job: embed_pipeline]
   │
   ▼
[BigQuery: feature_mart.* / property_embeddings (BQ VECTOR_SEARCH source)]
   ▲
   │
[Cloud Run Service: search-api]
   │  /search
   ├──→ Meilisearch (Cloud Run、BM25 + filter)
   ├──→ BigQuery VECTOR_SEARCH (semantic ANN)
   ├──→ RRF 融合
   ├──→ LightGBM LambdaRank (in-process or Cloud Run inference)
   └──→ Pub/Sub (ranking_log / search-feedback)
                │
                ▼
        [BigQuery: mlops.ranking_log / mlops.feedback_events]
                │
                ▼
        [Cloud Run Job: training_job → GCS モデル更新]
```

### 2.1 主要コンポーネント

- **search-api** (`app/`): Cloud Run 上の API。lexical = Meilisearch、semantic = BigQuery `VECTOR_SEARCH`、再学習判定 + feedback 連携
- **embed pipeline / training job** (`pipeline/`): Cloud Run Jobs。Vertex AI Pipelines は使わない (Phase 5 で導入)
- **データ・特徴量**: BigQuery が検索・特徴量・ログの中心。Dataform で feature table / view 管理 (= Phase 5 Feature Store の入力源)
- **認証・シークレット**: CI/CD = WIF、ランタイム SA = 分離、Meilisearch master key = Secret Manager → Cloud Run `--set-secrets` 注入

## 3. Secret Manager (必須習得要素)

Phase 4 で初導入する必須 4 段:

1. Secret 作成 (`gcloud secrets create meili-master-key`)
2. SA IAM bind (`roles/secretmanager.secretAccessor`)
3. Cloud Run `--set-secrets MEILI_MASTER_KEY=meili-master-key:latest`
4. app 側 pydantic-settings で `MEILI_MASTER_KEY` を読み取り

**Phase 4 から Phase 7 まで一貫して必須**。Phase 7 では External Secrets Operator が Secret Manager → K8s Secret を自動同期するが、Secret Manager 自体は同じ。

## 4. Phase 4 で扱わないもの (Phase 5 以降の責務)

| 技術領域 | 導入 phase |
|---|---|
| Vertex AI Pipelines (KFP v2) / Endpoint / Model Registry / Monitoring | **Phase 5** |
| **Vertex AI Feature Store (Feature Group / Feature View / Feature Online Store)** (training-serving skew 防止のため必須) | **Phase 5** |
| **Vertex Vector Search** (Phase 4 の BQ `VECTOR_SEARCH` を置換) | **Phase 5** |
| Cloud Composer / Managed Airflow Gen 3 (本線 orchestration、Phase 5 → 6 引き算境界) / BQML / Dataflow Flex Template / TreeSHAP Explainability / Monitoring SLO + burn-rate / Composer-managed BQ monitoring query | **Phase 6** |
| GKE Autopilot / KServe / Gateway API / External Secrets Operator / Workload Identity / GMP / HPA / IAP / NetworkPolicy / Helm provider | **Phase 7** |
| 全 Phase 教材対象外 (Agent Builder / Vizier / Model Garden / Gemini RAG / W&B / Looker Studio / Doppler) | 親 [`README.md` §1](../../../README.md) |

## 5. 不変ルール

- 題材は不動産ハイブリッド検索 (Phase 3 から継承)
- 中核 5 要素は Phase 3-7 で不変
- Phase 4 では **ベクトル検索は BigQuery `VECTOR_SEARCH`** (Phase 5 で Vertex Vector Search に置換)
- Secret Manager は必須習得
- Terraform を正とする
- Vertex 以降の責務は持ち込まない
- 設計思想 (Port/Adapter / `core → ports ← adapters` / DI) は **Phase 2 から継承して不変**、adapter 実装だけ差し替え
