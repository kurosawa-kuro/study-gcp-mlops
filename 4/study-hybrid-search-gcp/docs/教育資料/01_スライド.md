# GCP-ML入門

ハイブリッド検索を  
**Cloud Run + BigQuery** へ移す

---

## この Phase の主眼

- ローカル検索を GCP 実行基盤へ移す
- `Cloud Run` / `BigQuery` / `GCS` / `Pub/Sub` の役割分担を理解する
- `Secret Manager` を必須要素として扱う
- `Terraform` と `WIF` で手作業を減らす

---

## Phase 3 から何が変わるか

| 観点 | Phase 3 | Phase 4 |
|---|---|---|
| 実行基盤 | ローカル中心 | GCP 中心 |
| 検索 API | FastAPI | FastAPI on Cloud Run |
| データ基盤 | PostgreSQL 中心 | BigQuery 中心 |
| 成果物保存 | ローカル中心 | GCS |
| IaC | 最小 | Terraform |

---

## 変わらない中核

- `Meilisearch`
- `multilingual-e5`
- `LightGBM LambdaRank`

この Phase は、検索の考え方を変えるのではなく  
**実行基盤を GCP に差し替える** ことが主題。

---

## GCP 技術の役割

| 技術 | 役割 |
|---|---|
| `Cloud Run` | API / Job 実行 |
| `BigQuery` | 物件、特徴量、ログ、semantic search |
| `GCS` | モデル・成果物保存 |
| `Pub/Sub` | ログとフィードバック連携 |
| `Dataform` | 特徴量テーブル整備 |

---

## Secret Manager は必須

教材で必ず押さえる点:

- Meilisearch master key を平文で持たない
- Cloud Run へ secret 注入する
- アプリは設定経由で読む

```text
Secret Manager
  ↓
Cloud Run --set-secrets
  ↓
app settings
```

---

## WIF と Terraform の位置づけ

### WIF
- GitHub Actions から SA key なしで GCP 認証する

### Terraform
- Cloud Run、BigQuery、Pub/Sub、IAM をコードで管理する

> 「クラウドに置く」だけでなく、再現可能に構築するのがこの Phase の学習点。

---

## MLOps パイプラインの流れ

```text
raw data
  ↓
Dataform で特徴量生成
  ↓
embedding-job / training-job
  ↓
GCS に成果物保存
  ↓
search-api が検索を処理
  ↓
feedback を再学習へつなぐ
```

---

## どこを見るべきか

| パス | 見るポイント |
|---|---|
| `app/` | Cloud Run 上の API |
| `common/` | embedding / feature engineering 共通化 |
| `definitions/` | Dataform SQL |
| `infra/` | Terraform |
| `monitoring/` | skew チェック |

---

## この Phase で押さえること

1. Phase 3 の検索思想はそのまま
2. GCP では `Cloud Run + BigQuery + GCS` が主役
3. `Secret Manager` は必須
4. `Terraform + WIF` で運用の再現性を上げる
