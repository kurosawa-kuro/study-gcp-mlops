# 02_移行ロードマップ

本ドキュメントは Phase 4 (`study-hybrid-search-gcp`) の決定的仕様を管理する。Phase 4 は **Phase 5-7 の完成形から Vertex 標準 MLOps、PMLE 追加技術、GKE serving 差分を引いた、GCP serverless 実行基盤の学習フェーズ** とする。

目的は、Phase 3 のローカル不動産ハイブリッド検索を土台に、**Cloud Run / BigQuery / GCS / Pub/Sub / Terraform / WIF / Secret Manager** を使ったクラウド実行基盤へ移し替えること。

## 0. 不変ルール

- 題材は不動産ハイブリッド検索のまま維持する
- 検索中核は `Meilisearch + BigQuery VECTOR_SEARCH + multilingual-e5 + RRF + LightGBM LambdaRank`
- Secret Manager は必須習得要素として扱う
- IaC は Terraform を正とする
- Vertex 標準 MLOps や Phase 6 以降の追加技術は持ち込まない

## 1. 学習主眼

Phase 4 で新しく吸収させるのは次の GCP 基盤要素。

| 技術 | 学ぶ観点 |
|---|---|
| Cloud Run | API / job の実行基盤 |
| BigQuery | 特徴量、検索、ログ、再学習判定 |
| GCS | モデル成果物保管 |
| Pub/Sub | ranking log / feedback の非同期化 |
| Terraform | GCP リソースの IaC |
| Workload Identity Federation | CI/CD 認証 |
| Secret Manager | Meilisearch master key の安全な注入 |

## 2. 対象外

Phase 4 では以下を扱わない。

- Vertex AI Pipelines
- Vertex AI Endpoints
- Model Registry
- Feature Group
- Model Monitoring
- BQML
- Dataflow
- Monitoring SLO
- Explainable AI
- GKE / KServe / Gateway API
- 親 [`README.md` §1 教材対象外](../../../README.md) / [`docs/02_移行ロードマップ.md` §1.4](../../../docs/tasks/02_移行ロードマップ.md) の禁止技術

## 3. 実装方針

- Phase 3 の検索コアは維持し、実行基盤だけを GCP へ移す
- API は Cloud Run Service、学習と埋め込みは Cloud Run Jobs とする
- 特徴量は BigQuery + Dataform で管理する
- 認証は WIF と IAM を前提にする
- シークレットは Secret Manager から注入する

## 4. 修正方針

### 4.1 コード

- Vertex 以降の責務は入れない
- GCP serverless 実行基盤に必要な責務だけを残す
- Feature parity は維持する

### 4.2 文書

- README / CLAUDE / `docs/01_仕様と設計.md` は GCP 基盤移行を主題に統一する
- Phase 5 以降の技術は「次 Phase で追加するもの」としてのみ触れる
- 教育資料は Cloud Run / BigQuery / Terraform / Secret Manager / WIF を中心に整理する

### 4.3 検証方針

- 移行中は差分を意識しつつ、コード修正の速度を優先する
- E2E や CI/CD 上の総合検証は後段へ回す
- ローカルで回せる単体テスト、限定的な lint、局所的な確認を優先する
- 既存の lint / test 負債は、今回の移行差分と無関係なら別件として切り分ける
