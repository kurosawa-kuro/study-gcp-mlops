## 2. Phase 4 のスコープ (改訂版)

詳細仕様は [`01_仕様と設計.md`](01_仕様と設計.md)。要点:

- **題材**: 不動産ハイブリッド検索 (Phase 3 から継承)
- **主眼**: GCP MLOps の土台 (Cloud Run / Cloud Run Jobs / BigQuery / GCS / Secret Manager / Terraform / WIF / IAM)、軽量 retrain trigger (Cloud Scheduler + Eventarc + Cloud Function (Gen2))、BigQuery `VECTOR_SEARCH` ベースの semantic 検索
- **実行基盤**: uv + クラウド実行基盤
- **境界**: Vertex AI 以降は入れない (Phase 5 / 6 = 論理 Phase で資料化、Phase 7 = canonical で本実装)

### 2.1 Phase 3 → Phase 4 の足し算

- Cloud Run / Cloud Run Jobs (Docker Compose からの移行)
- BigQuery (PostgreSQL からの移行、`VECTOR_SEARCH` 含む)
- GCS (local filesystem からの移行)
- Secret Manager → Cloud Run secret injection (必須習得)
- Terraform (IaC)
- Workload Identity Federation (CI/CD 認証、SA Key 禁止)
- Cloud Logging + Cloud Monitoring
- Pub/Sub → BigQuery (BQ Subscription) によるログ送出
- Cloud Scheduler + Eventarc + Cloud Function (Gen2) による軽量 retrain trigger
- Artifact Registry + Cloud Build + GitHub Actions による CI/CD

### 2.2 Phase 4 で扱わないもの

- Vertex AI Pipelines / Feature Store / Vector Search / Model Registry / Monitoring (Phase 5 論理 Phase で資料化、Phase 7 で本実装)
- Cloud Composer (Phase 6 論理 Phase で資料化、Phase 7 で本実装)
- BQML / Dataflow / TreeSHAP / SLO + burn-rate (Phase 6 論理 Phase で資料化、Phase 7 で本実装)
- GKE / KServe (Phase 7 で本実装)

## 3. 不変ルール

- **Phase 7 完成版からの引き算ではなく Phase 3 からの足し算で構築**
- 題材は不動産ハイブリッド検索 (Phase 3 から継承)
- 中核 5 要素は Phase 3-7 で不変 (Phase 4 = BQ `VECTOR_SEARCH`)
- Secret Manager は必須習得
- Terraform を正とする
- **Vertex AI / Cloud Composer / GKE / KServe / PMLE 追加技術は持ち込まない**
- 設計思想 (Port/Adapter / `core → ports ← adapters` / DI) は Phase 2 から継承して不変
