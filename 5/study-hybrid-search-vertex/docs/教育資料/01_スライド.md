# Vertex AI 入門

Phase 4 の GCP 構成を  
**Vertex 標準 MLOps** に寄せる

---

## この Phase の主眼

- `Vertex AI Pipelines` で学習・埋め込みを管理する
- `Vertex Endpoint` へ推論を外出しする
- `Model Registry` でモデルの正本を持つ
- `Model Monitoring` と `Feature Group` を追加する

---

## Phase 4 から何が変わるか

| 観点 | Phase 4 | Phase 5 |
|---|---|---|
| 学習オーケストレーション | Cloud Run Jobs | Vertex AI Pipelines |
| 推論呼び出し | API 内部 load が中心 | Vertex Endpoint 呼び出し |
| モデル管理 | GCS + training_runs | Model Registry + training_runs |
| 監視 | Cloud Monitoring 中心 | Model Monitoring 追加 |

---

## 変わらない中核

- `Meilisearch`
- `BigQuery VECTOR_SEARCH`
- `RRF`
- `LightGBM LambdaRank`

Phase 5 でも、検索コアは置き換えない。  
**変えるのは MLOps の実行責務**。

---

## Vertex AI の役割分担

| 機能 | 役割 |
|---|---|
| `Vertex AI Pipelines` | 学習・埋め込み DAG |
| `Vertex Endpoint` | encoder / reranker 推論 |
| `Model Registry` | モデル版管理 |
| `Feature Group` | 特徴量管理の補助 |
| `Model Monitoring` | 監視と drift 検知 |

---

## search-api の役割変化

### Phase 4
- API が encoder や booster を強く抱える

### Phase 5
- API はプロキシ寄りになる
- 重い推論処理は Vertex Endpoint に委譲する

```text
search-api
  ↓
Vertex Endpoint encoder / reranker
```

---

## Port/Adapter の効きどころ

- core の検索ロジックは大きく変えない
- adapter だけ Vertex 実装へ差し替える
- だから「基盤差し替え」を学習テーマとして切り出せる

> この Phase は Port/Adapter の価値が最も見えやすい。

---

## 学習者が見るべきポイント

1. Pipeline に移した責務
2. Endpoint に移した責務
3. Registry に集約した責務
4. それでも残した `Meilisearch` と `BigQuery` の役割

---

## 教材で扱わないもの

- `W&B`
- `Doppler`

教材では、Vertex 標準機能と既存 `training_runs` を軸に説明する。

---

## この Phase で押さえること

1. Phase 4 の GCP 構成を Vertex 標準へ寄せる
2. 検索コアは変えず、MLOps の責務だけ動かす
3. `Pipelines` / `Endpoint` / `Registry` / `Monitoring` が新しい主役

---

## 次の Phase へ

Phase 6 では  
**BQML / Dataflow / SLO / Gemini RAG / Explainable AI** などを  
既存コードへ追加して PMLE 学習に広げる。
