## 2. DB テーブル一覧とリレーション

### テーブル一覧

| テーブル | 導入 Phase | 主な役割 |
|---|---|---|
| `properties` | Phase 0-1 | 物件マスタ（基本属性） |
| `property_stats` | Phase 2 | 物件ごとの行動集計（impression/click/CTR 等） |
| `search_logs` | Phase 2 | 検索セッションログ（クエリ・結果・フィードバック） |
| `property_features` | Phase 3 | ランキング用特徴量（日次更新） |
| `batch_job_logs` | Phase 3 | バッチ実行ログ（成功/失敗/件数） |
| `property_embeddings` | Phase 4 | 物件 ME5 埋め込みベクトル |
| `ranking_compare_logs` | Phase 5 | Meili 順 vs 再ランキング順の差分ログ |
| `offline_eval_reports` | Phase 6 | NDCG/MAP/Recall の評価結果スナップショット |
| `kpi_daily_stats` | Phase 6 | 日次 KPI（impressions/clicks/CTR/CVR 等） |
| `model_adoption_decisions` | Phase 6 | モデル採用/非採用の判定記録 |

| area | NUMERIC(8,2) | NOT NULL, >0 | 専有面積（㎡） |
| age | INTEGER | NOT NULL, ≥0 | 築年数 |
| walk_min | INTEGER | NOT NULL, ≥0 | 最寄り駅徒歩分 |
| pet | BOOLEAN | NOT NULL, DEFAULT FALSE | ペット可否 |
| is_active | BOOLEAN | NOT NULL, DEFAULT TRUE | 掲載中フラグ（Phase 3 追加） |

## 3. API エンドポイント

### `GET /health`

| 項目 | 内容 |
|---|---|
| 実装 | `app/main.py` |
| 説明 | ヘルスチェック |
| レスポンス | `{"status": "ok"}` |

---

### `GET /search`

| 項目 | 内容 |
|---|---|
| 実装 | `app/routes/search.py` |
| 説明 | 物件検索（`SearchPropertiesUseCase` を DI 注入して実行。内部で Redis キャッシュ確認 → Meili → ME5 再ランク → LightGBM 再ランク） |

**クエリパラメータ**

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| q | str | `""` | 全文検索クエリ |
| user_id | int | null | ユーザー ID（ログ紐付け用） |
| city | str | null | 市区フィルタ |
| layout | str | null | 間取りフィルタ（例: 2LDK） |
| price_lte | int | null | 賃料上限（円） |
| pet | bool | null | ペット可フィルタ |
| walk_min | int | null | 駅徒歩分上限 |
| limit | int | 20 | 返却件数（1〜100） |
| candidate_limit | int | 100 | Meili 取得候補数（1〜200） |

**レスポンス例**

```json
{
  "search_log_id": 42,
  "compare_log_id": 7,
  "count": 20,
  "items": [
    {
      "id": 1,
      "title": "...",
      "price": 85000,
      "me5_score": 0.074675,
      "lgbm_score": 0.357792,
      ...
    }
  ]
}
```

---

### `POST /feedback`

| 項目 | 内容 |
|---|---|
| 実装 | `app/routes/feedback.py` |
| 説明 | ユーザー行動ログ記録（`RecordFeedbackUseCase` を DI 注入して実行） |

**リクエストボディ**

```json
{
  "user_id": 1,
  "property_id": 5,
  "action": "favorite",
  "search_log_id": 42
}
```

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| user_id | int | 任意 | ユーザー ID |
| property_id | int | 必須 | 対象物件 ID |
| action | str | 必須 | `click` / `favorite` / `inquiry` |
| search_log_id | int | 任意 | 紐付ける検索ログ ID |

**レスポンス例**

```json
{
  "status": "ok",
  "property_id": 5,
  "action": "favorite",
  "search_log_updated": true
}
```

## 4. バッチジョブ一覧

| ファイル | make ターゲット | 実行契機 | 説明 |
|---|---|---|---|
| `ml/data/loaders/sync_properties_to_meilisearch.py` | `make ops-sync` | 物件データ変更時 | DB → Meilisearch 全件同期 |

| `ml/data/preprocess/generate_property_embeddings.py` | `make ops-embed` | 物件追加時・日次 | active 物件の ME5 embedding 事前計算 |
| `pipeline/batch/features/export_feature_report.py` | `make features-report` | デバッグ時 | 特徴量サマリ出力 |
| `pipeline/batch/evaluation/export_ranking_compare_report.py` | `make eval-compare` | 確認時 | Meili vs 再ランキング差分サマリ |
| `pipeline/batch/evaluation/run_offline_evaluation.py` | `make eval-offline` | 週次 | NDCG/MAP/Recall 計算・offline_eval_reports 保存 |

| `pipeline/batch/evaluation/export_weekly_evaluation_report.py` | `make eval-weekly-report` | 週次 | 週次評価レポート出力 + モデル採用判定 |
| `ml/training/model_builder.py` | `make ops-train-build` | 週次 | 学習用データセット CSV 生成（`artifacts/train/rank_train.csv`） |
| `ml/training/trainer.py` | `make ops-train-fit` | 週次 | LightGBM LambdaRank 学習 + モデル保存 |
| `ml/training/job.py` | `make ops-retrain` | 週次 | 採用判定に基づく再学習実行（build → fit を内包） |
| `pipeline/batch/maintenance/run_migrations.py` | `make *-migrate` | マイグレーション時 | SQL ファイルを DB に適用するユーティリティ |

### 運用チェックスクリプト一覧（scripts/ops）

| スクリプト | make ターゲット | 主用途 | 補足 |
|---|---|---|---|
| `scripts/local/ops/health_check.py` | `make ops-livez` | API ヘルスチェック | `/health` をリトライし、起動直後の一時失敗を吸収 |
| `scripts/local/ops/search_check.py` | `make ops-search` | 検索 API 疎通確認 | `/search` の結果 JSON を確認 |
| `scripts/local/ops/feedback_check.py` | `make ops-feedback` | フィードバック API 疎通確認 | `search_log_id` を取得して `POST /feedback` まで確認 |
| `scripts/local/ops/ranking_check.py` | `make ops-ranking` | 再ランキング結果の簡易確認 | `search_log_id`、先頭物件、`me5_score` を出力 |
| `scripts/local/ops/ranking_check_verbose.py` | `make ops-ranking-verbose` | 再ランキング結果の詳細確認 | 先頭物件の `lgbm_score`/`me5_score` を出力 |
| `scripts/local/ops/training_label_seed.py` | `make ops-label-seed` | 学習用ラベルシード投入 | `click/favorite/inquiry` の初期行動ログを投入 |
| `scripts/local/ops/training_fit_safe.py` | `make ops-train-fit-safe` | 安全な学習実行判定 | 学習データ件数/正例数が不足時は `ops-train-fit` を skip |

### 複合ターゲット（ops 系）

| make ターゲット | 実行内容 | 説明 |
|---|---|---|
| `make ops-bootstrap` | db-migrate-* → ops-sync → ops-embed → ops-label-seed → ops-train-build → ops-train-fit-safe | 全マイグレーション〜初回学習まで一括実行（学習は安全判定付き） |

| `make verify-pipeline` | check-layers → ops-livez → ops-search → ops-feedback → ops-ranking → ops-ranking-verbose → eval-compare → eval-offline | 代表 E2E 確認（API/ランキング/評価パイプライン） |
| `make api-refresh` | docker compose build api → up --force-recreate api | API コンテナの再ビルド＆再起動 |
