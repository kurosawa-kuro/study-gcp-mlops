# 02_移行ロードマップ

本ドキュメントは Phase 3 (`study-hybrid-search-local`) の決定的仕様を管理する。Phase 3 は **Phase 4-7 の完成形からクラウド実行基盤、Vertex 標準 MLOps、PMLE 追加技術、GKE serving 差分を引いた、ローカルハイブリッド検索の学習フェーズ** とする。

目的は、**Meilisearch / multilingual-e5 / Redis / LightGBM LambdaRank** を使った不動産ハイブリッド検索を、ローカル環境で一通り動かし、検索・特徴量・学習・評価の流れを掴むこと。

## 0. 不変ルール

- 題材は不動産ハイブリッド検索のまま維持する
- Local 検索中核は `Meilisearch + multilingual-e5 + Redis + LightGBM LambdaRank`
- `Meilisearch` は `Elasticsearch` より導入しやすいため採用する
- docker compose によるローカル完結を前提にする
- Cloud / Vertex / GKE 系の責務は持ち込まない

## 1. 学習主眼

Phase 3 で新しく吸収させるのは次のローカル検索基盤要素。

| 技術 | 学ぶ観点 |
|---|---|
| Meilisearch | lexical 検索 |
| multilingual-e5 | semantic 類似度 |
| Redis | キャッシュ |
| LightGBM LambdaRank | 再ランキング |
| PostgreSQL | ローカル永続化 |
| Docker Compose | ローカル実行基盤 |

## 2. 対象外

## 3. 実装方針

- ローカルで検索・学習・評価が一巡することを優先する
- lexical / semantic / rerank の 3 役を分けて学べる構成にする
- PostgreSQL / Redis / Meilisearch を docker compose で起動する
- Cloud 前提の責務は次 Phase に送る

## 4. 修正方針

### 4.1 コード

- ローカル実行に不要な Cloud 前提は入れない
- Meilisearch / Redis / PostgreSQL を前提とした説明に揃える

### 4.2 文書

- README / CLAUDE / `docs/01_仕様と設計.md` はローカルハイブリッド検索を主題に統一する
- Phase 4 以降の技術は「次 Phase で追加するもの」としてのみ触れる
- 教育資料は Meilisearch / ME5 / Redis / LambdaRank を中心に整理する

### 4.3 検証方針

- 移行中は差分を意識しつつ、コード修正の速度を優先する
- E2E や CI/CD 上の総合検証は後段へ回す
- ローカルで回せる単体テスト、限定的な lint、局所的な確認を優先する
- 既存の lint / test 負債は、今回の移行差分と無関係なら別件として切り分ける
