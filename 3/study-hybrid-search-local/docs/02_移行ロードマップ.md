## 1. 本ファイルの位置づけ

本 phase の **権威 1 位 (決定的仕様)** ファイル。Phase 3 (不動産ハイブリッド検索 Local) のスコープと再構築方針を定義する。

> **現状**: 本 phase の実装データ (コード) は 2026-05-02 に削除済 (`pre-phase3to6-impl-removal` tag で復元可)。改訂方針に基づき、コード化対象として再構築の対象。本 doc は仕様/教育資料として残置。

## 2. Phase 3 のスコープ

詳細仕様は [`01_仕様と設計.md`](01_仕様と設計.md)。要点:

- **題材**: 不動産ハイブリッド検索 (Phase 3-7 共通、ここから不動産検索ドメインに移行)
- **主眼**: lexical (Meilisearch BM25) + semantic (multilingual-e5) + RRF + LightGBM LambdaRank の 3 段検索を Local で一巡
- **実行基盤**: uv + Docker Compose
- **後続**: Phase 4 (GCP MLOps 土台) は本 phase からの足し算で構築する起点

## 3. 不変ルール

- 題材は不動産ハイブリッド検索 (Phase 3 / 4 / 7 のコード化対象 + Phase 5 / 6 論理 Phase 共通)
- 中核 5 要素: Meilisearch BM25 / multilingual-e5 / pgvector・簡易 ANN / RRF / LightGBM LambdaRank
- Cloud / Vertex / GKE 系の責務は持ち込まない (Phase 4 で GCP 土台を足し算、Vertex AI 以降は Phase 7 完成版コードに集約)
- 設計思想 (Port/Adapter / `core → ports ← adapters` / DI) は **Phase 2 から継承して不変**
