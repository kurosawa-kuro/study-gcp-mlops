# 01_仕様と設計

> 決定的仕様は `docs/02_移行ロードマップ.md` を正とする。本ファイルは Phase 3 の実装配置を説明する。

Phase 3 の主目的は、不動産ハイブリッド検索を **ローカル完結で一巡させる** こと。

## 0. 最重要ルール

- 不変なのは題材と Local 検索中核
- Local 検索中核は `Meilisearch + multilingual-e5 + Redis + LightGBM LambdaRank`
- 変えるのはローカルでの役割分担や実行導線の整理だけ

## 1. Phase 3 で扱うもの

| 技術 | 配置 |
|---|---|
| Meilisearch | lexical 検索 |
| multilingual-e5 | semantic 類似度 |
| Redis | キャッシュ |
| LightGBM LambdaRank | 再ランキング |
| PostgreSQL | 永続化 |
| Docker Compose | 実行基盤 |

## 2. 中核アーキテクチャ

1. Meilisearch で候補を取る
2. multilingual-e5 で意味類似度を補う
3. LightGBM LambdaRank で最終順位を決める
4. Redis で短時間キャッシュする
5. PostgreSQL にログや特徴量を保持する
