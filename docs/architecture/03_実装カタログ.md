## 使い分け

- 構成差分を追うとき: Phase 3 → 4 → 5 → 6 → 7 の順 (引き算戦略の **逆順** で読むと積み上げが見える)
- アプリ/API 実装を見るとき: Phase 2 / 3 / 4 / 5 / 6 の `app` 周辺を比較
- MLOps 基礎構成のみ見るとき: Phase 1 / 2 を先に参照
- PMLE 追加技術を実コード上で見るとき: Phase 6 の adapter / 追加エンドポイント / 追加 Terraform モジュール
- GKE + KServe serving + 完成版の実装カタログを見るとき: **Phase 7 (到達ゴール)**
