## 使い分け

- 構成差分を追うとき (足し算): Phase 1 → 2 → 3 → 4 の順
- 構成差分を追うとき (Phase 7 完成版を分解): Phase 7 → Phase 6 / 5 の論理 Phase 資料で技術境界を確認
- アプリ/API 実装を見るとき: Phase 2 / 3 / 4 / 7 の `app` 周辺を比較
- MLOps 基礎構成のみ見るとき: Phase 1 / 2 を先に参照
- Vertex AI MLOps 化を実コード上で見るとき: Phase 7 配下の `pipeline/` / `infra/terraform/modules/{vertex,vector_search}/`
- PMLE 追加技術を実コード上で見るとき: Phase 7 配下の Composer DAG (`pipeline/dags/`) / BQML / Dataflow / TreeSHAP 関連
- GKE + KServe serving + 完成版の実装カタログを見るとき: **Phase 7 (canonical 起点)**
