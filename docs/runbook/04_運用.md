## 運用観点の目安

- ローカル検証中心: Phase 1 / Phase 2 / Phase 3
- GCP 運用 (Cloud Run / BigQuery / Terraform / **Secret Manager**): Phase 4
- Vertex 連携運用 (Pipelines / Endpoints / Model Registry / Monitoring) + **Feature Store** + **Vertex Vector Search**: Phase 5 (orchestration は Phase 4 軽量経路を継続、Composer は未導入)
- Phase 5 継承 + **Composer 本線 orchestration** + PMLE 追加技術ラボ運用 (BQML / Dataflow / Monitoring SLO 等): Phase 6
- **GKE + KServe serving 運用 (到達ゴール、PDCA `make deploy-all` → `make run-all` → `make destroy-all`)**: Phase 7

## 共通事項

- 教材対象外技術は親 [`README.md` §1 教材対象外](../README.md) を参照 (canonical)
- **Secret Manager は Phase 4 で必須習得** (Meilisearch master key 題材)
- Phase 7 は GKE 側 Kubernetes Secret を **External Secrets Operator** で GCP Secret Manager から自動同期 (Phase 7 docs/05 §1, §3.5)
- 実験履歴は Phase 1-3 で `runs/{run_id}/`、Phase 4 以降で GCS / BigQuery / Vertex Pipelines Metadata / Vertex Model Registry
- Phase 横断の Composer DAG (`daily_feature_refresh` / `retrain_orchestration` / `monitoring_validation`) 起動・確認手順は Phase 7 docs/05 を参照 (Phase 6 で導入、Phase 7 で継承。Phase 5 では Composer 未導入で Phase 4 軽量経路を継続)
