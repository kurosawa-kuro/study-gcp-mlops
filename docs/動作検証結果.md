## 検証順序ポリシー (必須・Phase 5/6/7 共通)

ローカル検証チェックリスト (上から順に PASS させる):

1. **ビルド**: `make sync` (uv workspace 同期) + 当該 phase の build target (Docker compose build / image build) が成功すること
2. **疎通**: `make check` 相当 (ruff / ruff format --check / mypy strict / pytest) が **PASS**。少なくとも `make check-layers` で Port/Adapter 境界違反が 0 件

4. **アプリ E2E**: `make api-dev` (ローカル uvicorn) または Docker compose で API を起動し、`/livez` / `/readyz` が 200 を返すこと。Phase 7 なら追加で port-forward 経由 `/livez` `/search` `/feedback` を smoke
5. **検索三種動作**: lexical (Meilisearch BM25) / semantic (ME5 + BQ VECTOR_SEARCH or in-memory equivalent) / rerank (LightGBM LambdaRank) の **3 成分寄与がローカルで観測できること**。具体的には `make ops-search-components` 相当で `lexical_hits>0 ∧ semantic_hits>0 ∧ rerank_hits>0` を確認

**禁止事項**:

- 上記 5 点のいずれかが赤いまま `terraform apply` / `make deploy-all` / `make deploy-api` / `make deploy-kserve-models` を走らせない
- 「GCP 上でしか起きない」と推測しての盲目 deploy 禁止。再現できる最小ローカル fixture を先に作る
- ローカル PASS と GCP の差分を口頭で済ませず、本ドキュメントに `### ローカル前提` として実行ログ要約を残す

**例外**: GCP 側にしか存在しないリソース (Cloud Run service URL の存在確認 / IAM binding / WIF / Secret Manager 注入 / Gateway TLS Secret) を検証するときは、当然 GCP 側で実施する。ただしその場合も **ローカル 5 点 PASS が前提**であり、検証対象は「GCP 固有の配線」だけに絞る。

### Meilisearch 同期 (初期 5 件 + numeric 型厳守)

**前提**:
- user account に `roles/iam.serviceAccountTokenCreator` on `sa-api@mlops-dev-a.iam.gserviceaccount.com` が付与済
- `meili-search` Cloud Run service が real Meilisearch image で `Ready=True` (port 問題に注意)

**成功する curl シーケンス (2026-04-23 確定)**:
```bash
MEILI_URL=$(gcloud run services describe meili-search --project=mlops-dev-a --region=asia-northeast1 --format='value(status.url)')
TOKEN=$(gcloud auth print-identity-token --impersonate-service-account=sa-api@mlops-dev-a.iam.gserviceaccount.com --audiences="$MEILI_URL")

# 1) settings PATCH (searchable=title/city/ward/layout, filterable=rent/walk_min/age_years/layout/pet_ok)
curl -sS -X PATCH "$MEILI_URL/indexes/properties/settings" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"filterableAttributes":["rent","walk_min","age_years","layout","pet_ok"],"searchableAttributes":["title","city","ward","layout"]}'
# Expect: 202 {"taskUid":N,"status":"enqueued",...}

# 2) documents PUT — **rent / walk_min / age_years / pet_ok は numeric / bool 必須**
#    bq query --format=json は int64 を string に変換する仕様で、Meili filter `rent <= 150000` が壊れる。
#    だから Python で json.dumps する時に int/bool として渡す必要がある (bq CLI 直流しは NG)。
curl -sS -X PUT "$MEILI_URL/indexes/properties/documents" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  --data-binary @docs_numeric.json  # rent:120000 (int), pet_ok:true (bool) 等
# Expect: 202 → /tasks/N で status=succeeded を確認
```
