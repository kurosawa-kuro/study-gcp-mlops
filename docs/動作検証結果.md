## 成功ノウハウ抽出 (全 Phase 共通)

### Meilisearch 同期ベストプラクティス

- 前提: user に `roles/iam.serviceAccountTokenCreator` on `sa-api` 付与済
- BQ → Meilisearch は **integer 型を string に変換しない** (bq CLI `--format=json` の罠を回避)
- Cloud Run service は `MEILI_HTTP_ADDR=0.0.0.0:8080` env 必須
- `/search` 疎通は sa-api OIDC token 経由で確認

### Vertex Endpoint 効率化

- Model Registry 登録と Endpoint deploy は **並列実行可能** (独立操作)
- 学習リポは **n1-standard-2 min=1/max=1** (最小構成)
- quota `CustomModelServingCPUsPerProjectPerRegion` は Endpoint ごとの vCPU 合算で計算

### destroy-all / deploy-all サイクル

- **5 step destroy**: endpoint undeploy → bucket wipe → deletion_protection flip → terraform destroy
- destroy 所要時間: **~50-70 秒** (tfstate 依存)
- deploy 所要時間: **5-10 分** (terraform apply) + **20-25 分** (Endpoint deploy) = **25-35 分**

### 検索 E2E 成功条件

- `/livez 200` だけでは不十分
- **3 成分寄与 (`lexical_hits > 0 && semantic_hits > 0 && rerank_hits > 0`) が最小必要条件**
- 精度確認は **20 ケース NDCG@10 / HitRate@10 / MRR@10** で固定
