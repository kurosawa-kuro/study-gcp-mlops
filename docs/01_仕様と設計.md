## Phase 設計の軸

- Phase 1: **ML 基礎に集中** (学習・評価・保存)
- Phase 2: **App / Pipeline / Port-Adapter** を導入
- Phase 3-5: 不動産検索ドメインで **Local → GCP → Vertex AI** へ展開
- Phase 6: Phase 5 と**同じ不動産ハイブリッド検索ドメインを維持**し、PMLE 試験範囲の追加技術を Phase 5 実コードに統合
- Phase 7: **Phase 6 の serving 層を GKE + KServe に置き換える到達ゴール** (Vertex AI Pipelines / Feature Store / Vertex Vector Search / Cloud Composer orchestration / Model Registry / BigQuery / Meilisearch 等は Phase 6 から継承)

## 設計思想の不変性

- Phase 間のコードは原則共有しない (教材としての独立性を優先)
- ただし **設計思想 (Port/Adapter / 依存方向 / `core → ports ← adapters`) は一貫** させ、**adapter 実装だけ差し替える**
- 実行方式の段差 (Docker Compose → uv + クラウド → Vertex AI → GKE) は、同じ設計思想を保ったまま実行基盤だけを段階的に差し替える学習設計

## 共通非負制約

### 全 Phase 共通

- 教材対象外技術は導入しない (親 [`README.md` §1](../README.md))
- 設計思想 (Port/Adapter / 依存方向 / `core → ports ← adapters`) を破壊しない
- Phase 間のコード共有や広域リファクタは原則行わない

### Phase 3-7 (ハイブリッド検索系)

- ハイブリッド検索中核 5 要素を維持 (詳細は Phase 7 docs/01 §2.3)
- 検索品質改善はこの基本構成を前提に行う
- 置換・削減・無効化は事前の明示合意を必要とする

### Phase 6-7 (Composer orchestration)

詳細は §「Cloud Composer の位置づけ」(上記) と Phase 7 docs/01 §3 / §3.6 (カニバリ NG)。要点: Phase 6 で本線、Vertex `PipelineJobSchedule` は完全撤去、軽量経路は smoke / manual 用途で残置。

### Phase 6 追加 (PMLE 統合特有)

- 題材は Phase 5 と同じ不動産ハイブリッド検索ドメインを維持
- ハイブリッド検索中核コードの挙動は絶対に変えない
- 中核以外の改変は PMLE 学習のため自由に行う
- default feature flag では Phase 5 挙動を維持 (新技術は opt-in)
- `make check` / parity invariant / Port/Adapter 境界検知 / WIF は追加コードも含めて継続して PASS させる

### Phase 7 追加 (到達ゴール)

- Phase 6 の学習/データ基盤をそのまま継承し、serving 層のみ差し替える

## 教材対象外 (全 Phase 禁止)

詳細は親 [`README.md` §1 教材対象外](../README.md):

- Agent Builder / Vizier / Model Garden / Gemini RAG (機能カニバリ / 学習価値が低い)
- Vertex Vector Search は Phase 5 以降の semantic 検索の本番 serving index として採用 (embedding 履歴・メタデータの正本は BigQuery 側に残す)
- W&B / Looker Studio / Doppler

## 運用・成果物の置き場 (要点のみ)

詳細は Phase 7 [`docs/05_運用.md`](../7/study-hybrid-search-gke/docs/05_運用.md) と各 Phase 配下 docs:

- Phase 1-3: ローカル (`runs/{run_id}/` + git commit hash)
- Phase 4: GCS / BigQuery / Cloud Logging / Monitoring / GitHub Actions + WIF / Terraform / Secret Manager / **BigQuery feature table・view** (Phase 5 Feature Store の入力源)
- Phase 5: Vertex Model Registry / Pipelines / Endpoint / Model Monitoring / **Feature Store** + **Vertex Vector Search** (必須)
- Phase 6: Phase 5 継承 + **Cloud Composer 本線 + PMLE 追加技術**
- Phase 7: Phase 6 継承、serving 層のみ GKE + KServe に差し替え + KServe から Feature Online Store opt-in 参照
