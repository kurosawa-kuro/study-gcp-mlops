## Phase 設計の軸

- Phase 1: **ML 基礎に集中**（学習・評価・保存）
- Phase 2: **App / Pipeline / Port-Adapter** を導入
- Phase 3-5: 不動産検索ドメインで **Local → GCP → Vertex AI** へ展開
- Phase 6: Phase 5 と**同じ不動産ハイブリッド検索ドメインを維持**し、PMLE 試験範囲の追加技術を Phase 5 実コードに **adapter / 副経路 / 追加エンドポイント / 追加 Terraform** として統合
- Phase 7: **Phase 6 の serving 層を GKE + KServe に置き換える到達ゴール**（Pipelines / Feature Group / Model Registry / BigQuery / Meilisearch 等は Phase 6 から継承）

## 共通非負制約

### 全 Phase 共通

- 設計思想（Port/Adapter / 依存方向 / `core → ports ← adapters`）を破壊しない
- Phase 間のコードは原則共有しない（教材としての独立性を優先）
- adapter 実装の差し替えで実行基盤を切り替える（思想は一貫）

### Phase 3-7（ハイブリッド検索系）

- ハイブリッド検索の基盤は **LightGBM + multilingual-e5 + Meilisearch** を維持
- Phase 4 以降は **RRF 融合 + LightGBM LambdaRank 再ランク** を基本構成
- 置換・削減・無効化は事前に明示的な合意を必要とする

### Phase 6 追加（PMLE 統合特有）

- **題材は Phase 5 と同じ不動産ハイブリッド検索ドメインを維持**
- **ハイブリッド検索中核コード（Meilisearch BM25 + BQ `VECTOR_SEARCH` + multilingual-e5 + RRF + LightGBM LambdaRank）の挙動は絶対に変えない**
- **default feature flag では Phase 5 挙動を維持**（新技術は opt-in）
- 中核以外の改変は PMLE 学習のため自由に行う（新 Port / Adapter / Endpoint / KFP component / Terraform モジュール / parity 同 PR 更新）
- `make check` / parity invariant / Port/Adapter 境界検知 / WIF は追加コードも含めて継続して PASS させる

### Phase 7 追加（到達ゴール）

- **Phase 6** の学習/データ基盤をそのまま継承し、serving 層のみ GKE + KServe に差し替える

## 教材対象外（全 Phase 禁止）

- 🚫 **ハイブリッド検索中核とカニバリ・学習価値が低いため全 Phase で導入・言及しない**:
  - Vertex Vector Search
  - Agent Builder
  - Vizier
  - Model Garden
  - Gemini RAG
- 🚫 **W&B / Looker Studio / Doppler は教材対象外**（2026-04-24 決定、`docs/phase調整案.md` に沿って除去済）

## 必須習得（Phase 4 以降）

- **Secret Manager**: 題材は Meilisearch master key（Phase 4-7 横断で使える実在の秘匿情報）。Secret 作成 → SA IAM bind → Cloud Run `--set-secrets` 注入 → pydantic-settings 読み取り の 4 段を踏む

## 運用・成果物の置き場（段階移行）

- Phase 1-3: ローカル成果物（`model.pkl` / `metrics.json` / `runs/{run_id}/` + git commit hash）
- Phase 4: GCS / BigQuery / Cloud Logging / Cloud Monitoring / GitHub Actions + WIF / Terraform / **Secret Manager**
- Phase 5: Vertex Model Registry / Vertex Pipelines Metadata / Vertex Endpoint / Vertex Model Monitoring
- Phase 6: Phase 5 を継承 + 追加技術は adapter / 副経路 / 追加エンドポイント / 追加 Terraform として統合（default は Phase 5 挙動）
- Phase 7: Phase 6 から学習/データ基盤を継承、serving 層のみ GKE + KServe に差し替え（Meilisearch master key は Kubernetes Secret 手動同期で注入）
