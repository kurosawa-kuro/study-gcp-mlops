## 共通非負制約（Phase 3/4/5/6/7）

- ハイブリッド検索の必須構成は **LightGBM + multilingual-e5 + Meilisearch**
- Phase 4 以降は RRF 融合 + LightGBM LambdaRank 再ランクを基本構成とする
- Phase 5 は Vertex レイヤ後付けで、検索コアは Phase 4 から継承
- **Phase 6 は中核コード (`/search` default 挙動) を絶対に変えない**。PMLE 追加技術は adapter / 副経路 / 追加エンドポイント / 追加 Terraform として統合し、default flag では Phase 5 挙動を維持
- Phase 7 は **Phase 6** の学習/データ基盤を継承し、serving 層のみ GKE + KServe に差し替える

## 教材対象外（2026-04-24 決定）

- **W&B / Looker Studio / Doppler** は教材対象外。実装は `docs/phase調整案.md` に沿って除去済
- **Secret Manager は Phase 4 で必須習得**。題材は Meilisearch master key（Phase 4-7 横断で使える実在の秘匿情報）
