## 2. システムアーキテクチャ

```
[Cloud Run Job: ml-batch]
   ・Python 3.11 + scikit-learn + MLflow
   ・データ取得 → 学習 → 評価 → モデル保存 → ログ出力
   ・環境変数 GCS_BUCKET / JOB_NAME を注入
         |
         ├── モデル保存
         |       ↓
         |   [GCS: mlops-dev-a-data/models/]
         |       └── YYYYMMDD/model_YYYYMMDD_HHMMSS.pkl
         |
         └── ログ出力
                 ↓
             [GCS: mlops-dev-a-data/logs/]
                 └── YYYYMMDD/ml-batch_YYYYMMDD_HHMMSS.json

         ↑ イメージ取得
[Artifact Registry: mlops-dev-a-docker]
   ・Docker形式
   ・イメージ: ml-batch:latest

         ↑ 定期実行
[Cloud Scheduler: ml-batch-schedule]
   ・毎日 9:00 JST

         ↑ 自動デプロイ
[GitHub Actions: batch-deploy.yml]
   ・main push → test → build → push → Cloud Run Job更新

---（将来）---

[Cloud Run Service: ml-api]
   ・FastAPI
   ・GCSからモデルロード → 推論レスポンス

[BigQuery]（将来）
   ・features / predictions / metrics
```

## 5. アプリケーション設計

### batch (Cloud Run Job) — ML学習パイプライン

| 項目 | 値 |
|------|-----|
| ベースイメージ | python:3.11-slim |
| 依存ライブラリ | scikit-learn, pandas, google-cloud-storage, mlflow |
| エントリポイント | python main.py |
| イメージURI | asia-northeast1-docker.pkg.dev/mlops-dev-a/mlops-dev-a-docker/ml-batch:latest |

#### 環境変数

| 変数名 | 値 | 設定元 |
|--------|-----|--------|
| GCS_BUCKET | mlops-dev-a-data | Terraform (cloud_run_job.tf) |
| JOB_NAME | ml-batch | Terraform (cloud_run_job.tf) |
| N_ESTIMATORS | 100 | デフォルト（環境変数で上書き可） |
| MAX_DEPTH | 10 | デフォルト（環境変数で上書き可） |
| RANDOM_STATE | 42 | デフォルト（環境変数で上書き可） |
| TEST_SIZE | 0.2 | デフォルト（環境変数で上書き可） |
| MLFLOW_EXPERIMENT_NAME | california-housing | デフォルト（環境変数で上書き可） |

#### 処理フロー

```
1. MLflow実験名を設定、start_run()
2. California Housingデータを取得（イメージ内にキャッシュ済み）
3. train/test分割（デフォルト 8:2）
4. ハイパーパラメータをMLflowに記録
5. RandomForestRegressorで学習
6. 評価（RMSE, MAE）→ MLflowにメトリクス記録
7. MLflowにモデルをartifactとして記録
8. GCSにモデル保存: gs://{BUCKET}/models/YYYYMMDD/model_YYYYMMDD_HHMMSS.pkl
9. GCSにログ書き出し: gs://{BUCKET}/logs/YYYYMMDD/{JOB_NAME}_YYYYMMDD_HHMMSS.json
```

#### ログフォーマット

```json
{
  "job": "ml-batch",
  "status": "success",
  "metrics": {"rmse": 0.5445, "mae": 0.3663},
  "train_size": 16512,
  "test_size": 4128,
  "mlflow_run_id": "cf77a1f475a14f569a463032e575e561",
  "timestamp": "2026-03-25T21:18:59+00:00"
}
```

#### GCS認証

Cloud Runのデフォルトサービスアカウントによる自動認証（キー不要）。
IAMで `roles/storage.objectCreator` を付与済み。

## 7. ビルド・デプロイ設計

### デプロイフロー（冪等）

```
make deploy (= make batch-deploy)
  |
  ├── 1. tf-apply-repo    Artifact Registryを先行作成
  ├── 2. batch-build       Dockerイメージビルド
  ├── 3. batch-push        Artifact Registryへpush
  └── 4. tf-apply          全リソース反映（GCS, IAM, Cloud Run Job, Scheduler）
```

### CI/CDフロー（GitHub Actions）

```
main branch push（src/batch/** 変更時）or 手動実行
  |
  ├── 1. test       pytest実行
  └── 2. deploy     Docker build → push → gcloud run jobs update
```

### リセットフロー

```
make reset
  |
  ├── 1. tf-destroy        GCPリソース全削除
  ├── 2. Terraformクリーン  .terraform/, *.tfstate 削除
  ├── 3. ML成果物クリーン   outputs/, mlruns/, mlflow.db 削除
  └── 4. Dockerイメージ削除  ローカルイメージ削除
```

## 9. セキュリティ
