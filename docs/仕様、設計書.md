## 5. アプリケーション設計

### batch (Cloud Run Job)

| 項目 | 値 |
|------|-----|
| ベースイメージ | python:3.11-slim |
| 依存ライブラリ | google-cloud-storage |
| エントリポイント | python main.py |
| イメージURI | asia-northeast1-docker.pkg.dev/mlops-dev-a/mlops-dev-a-docker/ml-batch:latest |

#### 環境変数

| 変数名 | 値 | 設定元 |
|--------|-----|--------|
| GCS_BUCKET | mlops-dev-a-data | Terraform (cloud_run_job.tf) |
| JOB_NAME | ml-batch | Terraform (cloud_run_job.tf) |

#### 処理フロー

```
1. 環境変数 JOB_NAME を取得（デフォルト: ml-batch）
2. "Hello from Cloud Run Job: {JOB_NAME}" を標準出力
3. 環境変数 GCS_BUCKET を取得
4. JSON構造化ログを生成
   {
     "job": "ml-batch",
     "status": "success",
     "message": "Hello from Cloud Run Job: ml-batch",
     "timestamp": "2026-03-25T19:53:51+00:00"
   }
5. GCSに書き出し: gs://{BUCKET}/logs/YYYYMMDD/{JOB_NAME}_YYYYMMDD_HHMMSS.json
```

#### GCS認証

Cloud Runのデフォルトサービスアカウントによる自動認証（キー不要）。
IAMで `roles/storage.objectCreator` を付与済み。

## 7. ビルド・デプロイ設計

### デプロイフロー（冪等）

```
make deploy (= make batch-deploy)
  |
  ├── 1. tf-apply-repo    Artifact Registryを先行作成
  ├── 2. batch-build       Dockerイメージビルド
  ├── 3. batch-push        Artifact Registryへpush
  └── 4. tf-apply          全リソース反映（GCS, IAM, Cloud Run Job）
```

この順序により、以下の依存関係を解決:
- イメージpushにはArtifact Registryが必要
- Cloud Run Job作成にはイメージが必要

### リセットフロー

```
make reset
  |
  ├── 1. tf-destroy        GCPリソース全削除
  ├── 2. ローカルクリーン   .terraform/, *.tfstate 削除
  └── 3. Dockerイメージ削除  ローカルイメージ削除
```

### Makeコマンド一覧

| コマンド | 内容 |
|---------|------|
| make deploy | 全体冪等デプロイ |
| make reset | 全リソース削除 & クリーン |
| make batch-build | Dockerイメージビルド |
| make batch-push | ビルド & push |
| make batch-run | Cloud Run Job実行 |
| make batch-logs | 実行履歴確認 |
| make tf-init | Terraform初期化 |
| make tf-plan | 差分確認 |
| make tf-apply | 全リソース反映 |
| make tf-destroy | 全リソース削除 |
| make gcp-setup-apis | 必要なAPI一括有効化 |
| make gcp-setup-docker | Docker認証設定 |
