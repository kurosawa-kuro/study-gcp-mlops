# フォルダ・ファイル一覧

この文書は `/home/ubuntu/repos/study-gcp-mlops` 全体の「どこに何があるか」を 1 か所で確認するための索引です。  
目的は、phase ごとの境界は保ちつつ、参照時に複数フォルダを往復しなくて済むようにすることです。

## 使い方

- まず対象 phase を決める
- 次にその phase の `CLAUDE.md` / `README.md` / `Makefile` を見る
- 実装は `app` `ml` `pipeline` `scripts` を中心に追う
- 運用・設計は `docs/01_仕様と設計.md` `docs/03_実装カタログ.md` `docs/04_運用.md` を見る

4. `monitoring`
5. `infra`

### 変更時の原則

- phase をまたいで実装共有しない
- root docs は索引、phase docs は実務ソースとして扱う
- Phase 4-7 の `make` 公開 API は `scripts/setup` `scripts/deploy` `scripts/ops` を正本として追う
- `scripts/local` `scripts/dev` が残っている場合は、まず canonical 側があるか確認してから読む

## 補足

- `.venv` `.pytest_cache` `.ruff_cache` `.mypy_cache` `.terraform` `dist` `logs` `.cache` は生成物です
- この文書は「全ファイルの完全な dump」ではなく、「教材として迷わず辿れる構造索引」を目的にしています
- 実ファイル名の正確な一覧が必要な場合は、各 phase で `find <phase> -maxdepth 2` か `rg --files <phase>` を使うのが最短です
