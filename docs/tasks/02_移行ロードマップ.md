## 1. 教育コード化方針 (Phase 派生メタルール)

詳細は親 [`README.md` §4 コード化方針](../../README.md)。要点:

### 1.1 コード化対象 Phase (独立コード保守)

| Phase | 構築方針 |
|---|---|
| Phase 1 | ML 基礎の最小実装 (preprocess / training / evaluation / artifact 保存) |
| Phase 2 | App / Pipeline / Port-Adapter / DI を学ぶ最小実装 |
| Phase 3 | 不動産ハイブリッド検索の Local 完成形 |
| Phase 4 | **GCP MLOps の土台**。Phase 7 完成版からの引き算ではなく **Phase 3 からの足し算**で構築。Vertex AI 以降は含めない |
| Phase 7 | **canonical 起点 / 教材コード完成版**。全技術 (Cloud Composer + Vertex AI 一式 + GKE + KServe + PMLE 4 技術) を本 phase 配下に集約 |

### 1.2 論理 Phase (個別コード保守なし)

| Phase | 教育上の役割 | 説明方針 |
|---|---|---|
| Phase 5 | Vertex AI MLOps 化の技術境界 | Phase 7 完成版コードのうち Vertex AI Pipelines / Feature Store / Vector Search / Model Registry / Monitoring を抜き出して資料化 |
| Phase 6 | PMLE 追加技術 + Composer 本線化の論理境界 | Phase 7 完成版コードのうち BQML / Dataflow / Explainable AI / Monitoring SLO / Cloud Composer を抜き出して資料化 |

### 1.3 教材対象外 (全 Phase 禁止)

- 教材対象外技術 (Agent Builder / Vizier / Model Garden / Gemini RAG / W&B / Looker Studio / Doppler) の固有名は親 [`README.md` §1 教材対象外](../../README.md) の 1 箇所にのみ記載。それ以外の docs / コード / コメント / テスト / 履歴で言及禁止

### 1.4 旧「引き算戦略」との関係

旧版では「Phase 7 = canonical 起点、引き算で Phase 6 → 5 → 4 → 3 → 2 → 1」としていたが、本改訂で:

- **Phase 4 は Phase 3 からの足し算で独立コード化** (Phase 5 / 6 / 7 の前提となる GCP 土台を、Phase 7 からの引き算ではなく Phase 3 からの自然な足し算で学べる)
- **Phase 5 / 6 は論理 Phase 化** (中間コードを保守する代わりに、Phase 7 完成版コードを基準にして資料で技術境界を理解する)
- Phase 1 / 2 / 3 はコード化対象として独立保守を継続

これにより、差分コードの保守負担を避けつつ、各 Phase の技術習得主眼を段階的に説明できる構成にした。

## 2. Phase 7 の実装ロードマップ (Wave 1-3 仕様追従)

Phase 7 のコードを最新仕様 (Vertex Vector Search / Feature Store / Cloud Composer / GKE + KServe / 実案件 reference architecture) へ追従させるための具体実装計画は **Phase 7 [`docs/tasks/TASKS_ROADMAP.md`](../../7/study-hybrid-search-gke/docs/tasks/TASKS_ROADMAP.md) が正本**。

サマリ:

- **Wave 1** (検索アプリ層、ローカル完結) — PR-1 SemanticSearch Port + Vertex Vector Search adapter / PR-2 FeatureFetcher Port + FOS adapter / PR-3 VectorSearchWriter Port + embed pipeline 二重書き / PR-4 KServe Reranker × FOS opt-in。**M-Local 達成 (2026-05-01)**
- **Wave 2** (GCP インフラ層) — Vector Search Terraform module / FOS default 切替 / IAM / Composer 継承確認 / deploy 配線 / default flip
- **Wave 3** (docs 整合) — reference architecture / 実装カタログ / 運用 docs を Wave 1-2 完了に追従

詳細とチェックリストは Phase 7 docs/tasks/TASKS_ROADMAP を参照。
