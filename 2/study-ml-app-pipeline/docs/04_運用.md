# 04_運用

## 前提

- Docker + Docker Compose
- Python 3.10+

## 基本フロー

```bash
make build
make seed
make train
make serve
make test
```

## 各ジョブの実体

- `make seed` -> `python -m pipeline.seed_job.main`
- `make train` -> `python -m pipeline.train_job.main`
- `make serve` -> `uvicorn app.main:app`
- `make test` -> `pytest tests/`

## W&B 運用

- 設定キー:
  - wandb_api_key
  - wandb_project
  - wandb_dir
- 既定保存先: ml/wandb/wandb
- API キー未設定時: offline モードで ml/wandb/wandb 配下に run を保存

後から同期する例:

```bash
wandb sync ml/wandb/wandb/offline-run-XXXXXXXX
```

## 生成物

- モデル: ml/registry/artifacts/<run_id>/model.lgb
- メトリクス: ml/registry/artifacts/<run_id>/metrics.json
- 最新モデル参照: ml/registry/artifacts/latest
- 予測結果: PostgreSQL `prediction_runs`
- W&B ログ: ml/wandb/wandb

## クリーンアップ

```bash
make clean
```

削除対象:

- ml/registry/artifacts/
- ml/wandb/wandb/
