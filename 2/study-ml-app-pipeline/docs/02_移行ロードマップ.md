## 2. Phase 2 のスコープ

詳細仕様は [`01_仕様と設計.md`](01_仕様と設計.md)。要点:

- **題材**: California Housing 回帰 (Phase 1 から継承)
- **主眼**: FastAPI / lifespan DI / `core → ports ← adapters` / job 分離 (data_job / training_job / batch_serving_job)
- **学習器**: LightGBM 回帰 (Phase 1 から継承)
- **設計パターン**: Port / Adapter / DI を学ぶ起点 phase (Phase 3 以降も不変)
- **実行基盤**: Docker Compose + PostgreSQL

## 3. 不変ルール

- 題材は California Housing 回帰 (Phase 1 から継承)
- ML コアは LightGBM 回帰
- Phase 2 は App / Pipeline / Port-Adapter を学ぶフェーズ — 検索ドメインや GCP 責務は持ち込まない
- 設計思想 (Port/Adapter / `core → ports ← adapters` / DI) は **Phase 3 以降も不変** で、本 phase が起点
