# ナレーション要約（Phase 2 版）

## 導入

> Phase 2 では、Phase 1 の ML ロジックを維持したまま、アプリ層とパイプライン層を設計パターンで整理します。  
> キーワードは `core -> ports <- adapters` です。

## 差分の要点

> Phase 1 は学習処理の理解が中心でした。Phase 2 は「どう呼び出すか」「どう差し替えるか」を学びます。  
> FastAPI lifespan で DI し、`data_job` / `training_job` / `batch_serving_job` をジョブごとに分離します。

## Port/Adapter

> core は純粋ロジックだけを持ち、外部 I/O は ports で抽象化します。  
> adapters は PostgreSQL、filesystem の具体実装です。  
> 次のフェーズでは adapter を増やしても core を保てることが重要です。

## デモ要約

> `make seed` でデータ投入、`make train` で学習、`make serve` で API 起動。  
> `/predict` は container 経由で predictor を呼び出し、最新モデルで推論します。

## 締め

> Phase 2 の学びは、モデル自体よりも構造化と依存制御です。  
> この構造を前提に、Phase 3 では検索ドメインに拡張します。
