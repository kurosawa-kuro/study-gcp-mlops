# 01_仕様と設計

> 決定的仕様は `docs/02_移行ロードマップ.md` を正とする。本ファイルは Phase 2 の実装配置を説明する。

Phase 2 の主目的は、Phase 1 の California Housing 回帰を維持したまま、**App / Pipeline / Port-Adapter** を導入すること。

## 0. 最重要ルール

- 不変なのは題材と ML コア
- ML コアは California Housing + LightGBM 回帰
- 変えるのは API 入口、job 入口、依存方向、DI

## 1. Phase 2 で扱うもの

| 技術 | 配置 |
|---|---|
| FastAPI | `app/` |
| lifespan DI | `app/`, `ml/container.py` |
| Port-Adapter | `ml/ports/`, `ml/adapters/` |
| job 分離 | `pipeline/*_job/` |
| Filesystem / PostgreSQL adapter | `ml/adapters/` |

## 2. 中核アーキテクチャ

1. API リクエストや job 実行が inbound adapter になる
2. `ml/core` が純粋ロジックを持つ
3. `ml/ports` が抽象を定義する
4. `ml/adapters` が PostgreSQL や filesystem を実装する
5. `ml/container.py` が依存を配線する
