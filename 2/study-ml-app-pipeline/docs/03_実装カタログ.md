# 03_実装カタログ

Phase 2 の実装は `core -> ports <- adapters` の依存方向で構成する。

## common

- `common/config.py`: `BaseAppSettings`（`env/config/setting.yaml` + `env/secret/credential.yaml` 読み込み）
- `common/logging.py`: 共通ロガー
- `common/run_id.py`: `run_id` 生成

## ml/core

- `ml/core/schema.py`: `FEATURE_COLS` / `ENGINEERED_COLS` / `MODEL_COLS` / `TARGET_COL`
- `ml/core/preprocess.py`: 欠損補完・外れ値キャップ・`log1p` 変換
- `ml/core/feature_engineering.py`: `BedroomRatio` / `RoomsPerPerson` 生成
- `ml/core/evaluation.py`: RMSE/R2 算出・metrics 保存
- `ml/core/trainer.py`: LightGBM 学習・予測

## ml/ports

- `ml/ports/dataset.py`: `DatasetReader` Protocol（`load`/`write`/`write_predictions`）
- `ml/ports/model_store.py`: `ModelStore` Protocol（`save`/`load`/`set_latest`）
- `ml/ports/tracker.py`: `ExperimentTracker` Protocol（`start`/`log_metrics`/`finish`）

## ml/adapters

- `ml/adapters/postgres_dataset.py`: PostgreSQL への読み書き
- `ml/adapters/filesystem_model_store.py`: `ml/registry/artifacts/{run_id}` 保存・`latest` 更新
- `ml/adapters/wandb_tracker.py`: W&B online/offline ログ送信

## DI / App / Pipeline

- `ml/container.py`: `build_container(settings) -> Container`
- `app/main.py`: FastAPI lifespan で container を構築し latest モデルをロード
- `app/services/prediction_service.py`: 前処理 + 特徴量生成 + 推論
- `pipeline/seed_job/main.py`: sklearn から seed データ投入
- `pipeline/train_job/main.py`: 学習ジョブ（load -> preprocess -> engineer -> train -> save/log）
- `pipeline/predict_job/main.py`: バッチ予測ジョブ（latest モデルで推論し DB へ書き戻し）

## scripts / infra

- `scripts/core.py`: docker compose 実行共通化
- `scripts/local/setup/{seed,train}.py`: seed/train 起動
- `scripts/local/deploy/serve.py`: API 起動
- `scripts/local/ops/{test,clean}.py`: テスト・クリーン
- `infra/run/jobs/trainer/Dockerfile`: seed/train/predict 用イメージ
- `infra/run/services/api/Dockerfile`: API 用イメージ
