## 0. 前提と非負ルール (作業前に必ず確認)

- **中核 5 要素は不変**: Meilisearch BM25 / multilingual-e5 / ベクトルストア (Phase 4 = BQ `VECTOR_SEARCH` / Phase 5+ = Vertex AI Vector Search) / RRF / LightGBM LambdaRank
- **default `/search` 挙動は壊さない**: 切替は環境変数 (Strangler パターン)、既定は現行挙動
- **embedding 生成履歴・メタデータの正本は BigQuery 側**: Vertex Vector Search は本番 serving index、source は BQ embedding テーブル (`feature_mart.property_embeddings`)
- **Feature Store (Phase 5 必須)** を Phase 7 でも継承: training-serving skew 防止のため、Feature Online Store 経路を Phase 7 固有 opt-in で持つ
- **Meilisearch / Redis cache は据え置き**: 実案件 reference architecture (Elasticsearch + Redis 同義語辞書) は本 phase でもコードに混入させない (本 phase docs/01 §4 参照)
- **Feature parity invariant 6 ファイル原則** は SemanticSearch / FeatureFetcher 変更でも継続 PASS
- **Cloud Composer は Phase 5 で導入、Phase 6 で PMLE DAG 増設、Phase 7 で継承**: Phase 5 で `daily_feature_refresh` / `retrain_orchestration` / `monitoring_validation` の 3 本 DAG が orchestration 本線になる (Cloud Scheduler / Eventarc / Cloud Function trigger / Vertex `PipelineJobSchedule` は Phase 5 で撤去 = Phase 4 → 5 引き算境界)。Phase 6 で PMLE step を増設、Phase 7 はそのまま継承し、orchestration 二重化を作らない (詳細は親 [`README.md` §「Cloud Composer の位置づけ」](../../../README.md))。Wave 2 GCP インフラの中で Composer 関連 (Composer 環境 Terraform 継承確認 + DAG deploy) も含む

## 2. 移行戦略

### 2.1 Strangler パターン

既存 `BigQuerySemanticSearch` adapter を残したまま `VertexVectorSearchSemanticSearch` を新規追加。composition root が環境変数で切替。検証完了後に default を `vertex_vector_search` に切替、最終的に BQ adapter は Phase 4 専用として位置付ける (Phase 7 では deprecated 注釈、削除はしない)。

### 2.2 PR 分割粒度 (1 PR = 1 Port 原則)

| PR | スコープ | 受け入れ条件 |
|---|---|---|
| PR-1 | `SemanticSearch` Port + Vertex Vector Search adapter (app 層) + fake / unit test | `SEMANTIC_BACKEND=vertex_vector_search` で `/search` が in-memory fake 経由で 200 を返す |
| PR-2 | `FeatureFetcher` Port + Feature Online Store adapter (app 層) + fake | `FEATURE_FETCHER_BACKEND=online_store` で ranking が fake 経由で動作 |
| PR-3 | `VectorSearchWriter` Port + adapter (pipeline 層) + embed pipeline 二重書き | `pipeline/data_job/main.py` がローカルで BQ + fake VVS の両方に書く |
| PR-4 | KServe Reranker → Feature Online Store opt-in 参照 (Phase 7 固有) | `KSERVE_FEATURE_ONLINE_URL` 設定時のみ feature が VVS 経路で取得される |
| Wave 2 → | Terraform / IAM / deploy | 別 roadmap section §4 参照 |

### 2.3 default 切替の段取り

```
Step A: 全 PR merge 直後 → default = bq / inline (現状維持)
Step B: Wave 2 完了で GCP 側 index endpoint provision OK 後 → default = vertex_vector_search に flip
Step C: Step B が dev で 1 週間安定したら本 phase docs/01 §「実装段差」表を「Vertex Vector Search 以外撤去」に更新
```
