Phase 6 から継承した **7 件の恒久対処ギャップ** (Phase 7 でも有効):

1. `terraform destroy` の `deletion_protection=true` ブロック → `enable_deletion_protection` var + `scripts/setup/destroy_all.py` の `PROTECTED_TABLE_TARGETS` (10 table) で state-flip apply → destroy
2. 半 destroy 後の SA / Dataset IAM orphan → `-target` で apply 範囲を絞り recreate を回避
3. WIF pool/provider 30 日 soft-delete → `scripts/setup/deploy_all.py::_recover_wif_state` が undelete + `terraform import` を自動実行
4. `sa-api` が `feature_mart` 権限不足で /search が 403 → `infra/terraform/modules/data/main.tf` に `api_feature_viewer` (bigquery.dataViewer) 配線

6. Cloud Run `/healthz` が GFE に予約 → app に `/livez` alias を register、ops script は `/livez` を叩く (Phase 7 GKE 側でも踏襲)
7. KServe InferenceService の `storageUri` 差し替え — Vertex Model Registry `production` alias を `scripts/deploy/kserve_models.py` が `kubectl patch` で書き換え、`lifecycle.ignore_changes` は Terraform 側で不要 (manifest 管理が分離)

## 4. API エンドポイント

search-api (GKE Deployment、namespace `search`) が公開する HTTP ルート。`app/main.py::create_app()` で 4 つの surface (App API / Probes / Prom exposition / Operator UI) を **disjoint な path prefix** で配線する設計 — `/metrics` は Prometheus 専有、operator UI は `/ui/` 配下に閉じ込め、app の business API には prefix を付けない。`/` は 308 で `/ui/` に redirect。

認証境界は Gateway 側 IAP に集約 (`infra/manifests/policies/search-api-iap-policy.yaml` の `BackendConfig` で OIDC gate)。app 層は IAP 通過済前提で動き、handler に auth dependency は持たない。`infra/manifests/policies/search-api-networkpolicy.yaml` で egress を `kserve-inference` ns に絞っているため、外向き HTTP は KServe / Cloud Run `meili-search` / Vertex AI / GCP API のみ。

router 配線 (`app/main.py:74-92`):

```
include_router(health_router)        # /livez /healthz /readyz
include_router(search_router)        # /search
include_router(rag_router)           # /rag
include_router(feedback_router)      # /feedback
include_router(retrain_router)       # /jobs/check-retrain
include_router(model_router)         # /model/metrics /model/info        (prefix="/model")
include_router(build_ui_router(...), prefix="/ui")  # /ui/ /ui/model/metrics /ui/data
@app.get("/")  -> RedirectResponse("/ui/", 308)
app.mount("/static", StaticFiles(...))               # /static/*
Instrumentator(...).expose(endpoint="/metrics")      # /metrics
```

| GET | `/ui/data` | `templates/data.html` | seed / index / 状態確認の補助ビュー |
| GET | `/static/*` | `app/static/` (`StaticFiles`) | UI 用 CSS / JS / 画像 |

`infra/manifests/policies/search-api-iap-policy.yaml` の OIDC gate を通過した human user 向けで、IAP audience は GKE Gateway。production 公開時はここを社内ドメインに絞る。

### 4.5 Pydantic schema 一覧

| schema file | 主要型 |
|---|---|
| `app/schemas/search.py` | `SearchFilters` / `SearchRequest` / `SearchResultItem` (`property_id` / `lexical_rank` / `semantic_rank` / `final_rank` / `me5_score` / `score` / `attributions: dict[str,float] \| None` / `popularity_score: float \| None`) / `SearchResponse` / `FeedbackRequest` / `FeedbackResponse` |
| `app/schemas/rag.py` | `RagRequest` / `RagResponse` (`answer` / `cited_property_ids` / `search_results`) |
| `app/schemas/model.py` | `ModelMetricsResponse` (`summary: AccuracySummary` + `per_case: list[CaseMetric]`) / `ModelInfoResponse` |

`SearchResultItem.score` / `attributions` / `popularity_score` は **rerank / explain / BQML popularity が有効なときのみ non-null**。CLI smoke (`scripts/ops/search_components.py`) は 3 成分すべて non-null を要求する strict gate。
