### Phase 6 → Phase 7 の構造差分

- **新設**:
  - GKE / KServe 移行: `infra/terraform/modules/{gke,kserve}/`、`infra/manifests/{search-api,kserve,policies}/`、`scripts/deploy/{api_gke,kserve_models,monitor}.py`
  - DI 整備: `app/observability.py`、`app/container/{infra,search,ml,internal/optional_adapter}.py`、`app/settings/api.py` (flat + nested view)、`app/services/adapters/internal/{kserve_common,pubsub_diagnostics}.py`、`app/services/noop_adapters/` (旧 `fakes/`)
  - その他: `infra/sql/monitoring/` (root `monitoring/` から移送)、`scripts/ops/vertex/` (旧 flat `vertex_*.py` を sub-dir 化)、`tests/_fakes/` (旧 `tests/fakes/` を rename、`_` prefix で test 用 internal を明示)、`sa-external-secrets` (ESO 用 Terraform IAM)
- **撤去 / 役割変更**:
  - Cloud Run `search-api` → GKE Deployment (旧 `runtime/` module は `messaging/` に rename + Pub/Sub / Scheduler に責務縮小)
  - Vertex Endpoint `property-encoder/reranker` → KServe InferenceService (Terraform `modules/vertex/` の endpoint リソースは shell のみ残置、`modules/kserve/` が Helm chart 投入)
  - `scripts/local/` `scripts/dev/` ラッパー階層を完全撤去 (canonical = `scripts/{setup,deploy,ops,bqml,ci,sql}`)

## 4. API エンドポイント

search-api (GKE Deployment、namespace `search`) が公開する HTTP ルート。`app/main.py::create_app()` で 4 つの surface (App API / Probes / Prom exposition / Operator UI) を **disjoint な path prefix** で配線する設計 — `/metrics` は Prometheus 専有、operator UI は `/ui/` 配下に閉じ込め、app の business API には prefix を付けない。`/` は 308 で `/ui/` に redirect。

認証境界は Gateway 側 IAP に集約 (`infra/manifests/policies/search-api-iap-policy.yaml` の `GCPBackendPolicy` で OIDC gate)。app 層は IAP 通過済前提で動き、handler に auth dependency は持たない。`infra/manifests/policies/search-api-networkpolicy.yaml` で egress を `kserve-inference` ns に絞っているため、外向き HTTP は KServe / Cloud Run `meili-search` / Vertex AI / GCP API のみ。

router 配線 (`app/main.py`):

```
include_router(health_router)        # /livez /healthz /readyz
include_router(search_router)        # /search
include_router(feedback_router)      # /feedback
include_router(retrain_router)       # /jobs/check-retrain
include_router(model_router)         # /model/metrics /model/info        (prefix="/model")
include_router(build_ui_router(...), prefix="/ui")  # /ui/ /ui/model/metrics /ui/data
@app.get("/")  -> RedirectResponse("/ui/", 308)
app.mount("/static", StaticFiles(...))               # /static/*
Instrumentator(...).expose(endpoint="/metrics")      # /metrics
```

### 5.2 Terraform module 呼び出し順序

`infra/terraform/environments/dev/main.tf` の module 呼び出し依存:
