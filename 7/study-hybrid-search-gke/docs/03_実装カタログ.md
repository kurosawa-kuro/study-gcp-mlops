Phase 6 から継承した **7 件の恒久対処ギャップ** (Phase 7 でも有効):

1. `terraform destroy` の `deletion_protection=true` ブロック → `enable_deletion_protection` var + `scripts/setup/destroy_all.py` の `PROTECTED_TABLE_TARGETS` (10 table) で state-flip apply → destroy
2. 半 destroy 後の SA / Dataset IAM orphan → `-target` で apply 範囲を絞り recreate を回避
3. WIF pool/provider 30 日 soft-delete → `scripts/setup/deploy_all.py::_recover_wif_state` が undelete + `terraform import` を自動実行
4. `sa-api` が `feature_mart` 権限不足で /search が 403 → `infra/terraform/modules/data/main.tf` に `api_feature_viewer` (bigquery.dataViewer) 配線
5. `terraform apply` が Deployment env を reconcile → Phase 7 では Deployment manifest 側で env 差し替えを許容 (Terraform は cluster / KServe Helm のみ管理、manifest は `kubectl apply -k` で別管理)
6. Cloud Run `/healthz` が GFE に予約 → app に `/livez` alias を register、ops script は `/livez` を叩く (Phase 7 GKE 側でも踏襲)
7. KServe InferenceService の `storageUri` 差し替え — Vertex Model Registry `production` alias を `scripts/deploy/kserve_models.py` が `kubectl patch` で書き換え、`lifecycle.ignore_changes` は Terraform 側で不要 (manifest 管理が分離)

## 4. API エンドポイント

search-api (GKE Deployment、namespace `search`) が公開する HTTP ルート。

| メソッド | パス | 役割 |
|---|---|---|
| GET | `/livez` | 常時 OK (liveness)。`/healthz` も alias として register されているので local `make api-dev` 時はどちらでも届く |
| GET | `/readyz` | `candidate_retriever` + `encoder_client` がロード済か (readiness、503 if null)。成功時 `{status, search_enabled, rerank_enabled, model_path}` |
| POST | `/search` | `SearchRequest` → `SearchResponse`。`KServeEncoder.embed(query, "query")` で query vector 取得 → `BigQueryCandidateRetriever` で RRF 融合候補取得 → rerank 有効時は `KServeReranker` で score 付与。query param `?explain=true` (Phase 6 T4) / `?lexical=agent_builder` (Phase 6 T7) 対応 |
| POST | `/feedback` | `FeedbackRequest` → `FeedbackResponse`。Pub/Sub 経由で `mlops.feedback_events` へ |
| POST | `/jobs/check-retrain` | 再学習条件評価 → 条件成立時 `retrain-trigger` へ publish。ranker 指標 (NDCG / feedback rows) のみを見る |
| POST | `/rag` | **Phase 6 T6**: 内部 `run_search` → top-N を Gemini に要約させ `RagResponse` を返す。`enable_rag=false` で 503 |

Pydantic schema: `SearchFilters` / `SearchRequest` / `SearchResultItem` (`score` / `attributions` / `popularity_score` すべて rerank / explain / T1 有効時のみ non-null) / `SearchResponse` / `FeedbackRequest` / `FeedbackResponse` / `RagRequest` / `RagResponse`。
