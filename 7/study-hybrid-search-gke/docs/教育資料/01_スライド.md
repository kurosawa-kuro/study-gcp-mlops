# GKE + KServe 入門

Phase 6 の学習基盤を継承し、  
**serving 層だけを差し替える**

---

## この Phase の主眼

- `Cloud Run + Vertex Endpoint` だった serving 層を
  `GKE + KServe` に置き換える
- 学習基盤、データ基盤、検索中核は維持する
- Kubernetes 側の構成要素を学ぶ

---

## Phase 6 から何が変わるか

| 変わらないもの | 変わるもの |
|---|---|
| Meilisearch | `search-api` の配置先 |
| BigQuery | encoder / reranker の呼び出し先 |
| Vertex Pipelines | Gateway / Service / HPA |
| Model Registry | KServe InferenceService |

---

## 差し替えの考え方

```text
Phase 6
  search-api -> Vertex Endpoint

Phase 7
  search-api -> KServe InferenceService
```

**学ぶ対象は serving 差分**。  
検索アルゴリズムや学習方式を変える Phase ではない。

---

## GKE / KServe で見る技術

| 技術 | 役割 |
|---|---|
| `GKE Autopilot` | クラスタ実行基盤 |
| `KServe` | encoder / reranker の推論 serving |
| `Gateway API` | 外部入口 |
| `HPA` | API の水平スケール |
| `Workload Identity` | GCP 認証連携 |

---

## 変わらない中核

- `LightGBM`
- `multilingual-e5`
- `Meilisearch`
- `BigQuery VECTOR_SEARCH`
- `/search` の基本的な意味

> ここを変えないことで、学習者は「serving 差分」に集中できる。

---

## 学習者が見るべきポイント

1. API 本体は何を維持しているか
2. Endpoint 呼び出しが KServe 呼び出しにどう置き換わるか
3. Gateway / Deployment / InferenceService の責務分担
4. Vertex 時代に残すものと Kubernetes 側で新しく増えるもの

---

## 教材で強調すること

- Phase 7 は全面刷新ではない
- Phase 6 の成果を活かした serving 差分フェーズ
- Kubernetes 学習の入口として必要最小限を扱う

---

## この Phase で押さえること

1. 学習基盤とデータ基盤は継承する
2. serving 層だけを GKE + KServe に差し替える
3. GKE の学習対象は `Autopilot` / `Gateway API` / `KServe`

---

## このシリーズ全体の見方

1. Phase 1: ML 基礎
2. Phase 2: App / Pipeline / Port-Adapter
3. Phase 3: ハイブリッド検索 Local
4. Phase 4: GCP 実行基盤
5. Phase 5: Vertex 標準 MLOps
6. Phase 6: PMLE 追加技術統合
7. Phase 7: GKE + KServe serving 差分
