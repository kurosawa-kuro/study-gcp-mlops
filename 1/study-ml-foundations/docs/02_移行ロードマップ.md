# 02_移行ロードマップ

本ドキュメントは Phase 1 (`study-ml-foundations`) の決定的仕様を管理する。Phase 1 は **Phase 2-7 の完成形から API、Port-Adapter、検索ドメイン、クラウド基盤、Vertex 標準 MLOps、PMLE 追加技術、GKE serving 差分を引いた、ML 基礎の最小フェーズ** とする。

目的は、California Housing 回帰を題材に、**データ投入 / 前処理 / 特徴量生成 / 学習 / 評価 / 成果物保存** を一通り学ぶこと。

## 0. 不変ルール

- 題材は California Housing 回帰
- 学習器は LightGBM 回帰
- 評価は RMSE / R2
- 成果物は `run_id` / `model.lgb` / `metrics.json` を残す
- API や設計パターンは持ち込まない

## 1. 学習主眼

| 技術 | 学ぶ観点 |
|---|---|
| LightGBM | 回帰学習 |
| scikit-learn | データ取得補助 |
| numpy | 評価計算 |
| PostgreSQL | ローカル保存先 |
| Docker Compose | ローカル実行基盤 |

## 2. 対象外

- FastAPI
- lifespan DI
- Port-Adapter
- job 分離の設計論
- Meilisearch / multilingual-e5 / Redis / LambdaRank
- 不動産検索ドメイン
- Cloud Run / BigQuery / GCS / Pub/Sub
- Terraform / WIF / Secret Manager
- Vertex AI / BQML / Dataflow / RAG / Explainable AI / GKE
- W&B

## 3. 実装方針

- ML コアを最短で理解できる構成に絞る
- 学習と評価の流れを追いやすくする
- 成果物管理はローカルファイル中心にする

## 4. 検証方針

- 移行中は差分を意識しつつ、コード修正の速度を優先する
- E2E や CI/CD 上の総合検証は後段へ回す
- ローカルで回せる単体テスト、限定的な lint、局所的な確認を優先する
