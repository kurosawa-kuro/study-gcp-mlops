# 機械学習入門

California Housing を題材に  
**ML の基本工程** を学ぶ

---

## この Phase の主眼

- ML の流れを 1 本通して理解する
- `LightGBM` を使った回帰学習を体験する
- 前処理、特徴量、評価、成果物保存の関係をつかむ

> API や設計パターンはまだ扱わない。Phase 1 は ML コアに集中する。

---

## 今日のアジェンダ

| 項目 | 見るポイント |
|---|---|
| データ | 何を入力にするか |
| 前処理 | 学習しやすい形へどう整えるか |
| 特徴量 | 予測に効く情報をどう作るか |
| 学習 | `LightGBM` が何を学ぶか |
| 評価 | `RMSE` / `R²` をどう読むか |
| 保存 | `run_id` / `metrics.json` / model の残し方 |

---

## 機械学習とは

- 人がルールを全部書く代わりに、データからパターンを学ばせる
- 本 Phase では「住宅の特徴から価格を予測する」問題を扱う

```text
入力データ → 学習 → モデル → 予測
```

---

## このプロジェクトの入出力

| 入力 | 例 |
|---|---|
| 世帯収入 | `MedInc` |
| 築年数 | `HouseAge` |
| 平均部屋数 | `AveRooms` |
| 緯度・経度 | `Latitude`, `Longitude` |

出力:

- 住宅価格の予測値

---

## 前処理でやること

- 欠損値補完
- 外れ値キャップ
- 対数変換

```python
df = preprocess(df)
```

LightGBM は木ベースなのでスケーリングが主役ではない。  
この Phase では、**分布を壊しすぎない前処理** を学ぶ。

---

## 特徴量エンジニアリング

既存列を組み合わせて、モデルが学びやすい情報を作る。

| 生成特徴量 | 意味 |
|---|---|
| `BedroomRatio` | 寝室比率 |
| `RoomsPerPerson` | 1 人あたりの部屋数 |

```python
df["BedroomRatio"] = df["AveBedrms"] / df["AveRooms"]
df["RoomsPerPerson"] = df["AveRooms"] / df["AveOccup"]
```

---

## 学習と評価

### 学習
- `LightGBM` で回帰モデルを作る

### 評価
- `RMSE`: 誤差の大きさ
- `R²`: 説明できている割合

> 学習で作るのはモデル、評価で確認するのはそのモデルがどれだけ使えるか。

---

## 訓練データとテストデータ

- 訓練データ: モデルを学習させる
- テストデータ: 学習後に性能を確認する

```python
train_df = df.sample(frac=0.8, random_state=42)
test_df = df.drop(train_df.index)
```

テストデータを学習に混ぜないことが、再現性と妥当性の基本。

---

## 成果物の見方

| 成果物 | 役割 |
|---|---|
| `model.lgb` | 学習済みモデル |
| `metrics.json` | 評価結果 |
| `run_id` | 実行単位の識別子 |
| `latest` | 最新 run への参照 |

### 重要
- 実験管理の主役はローカル成果物
- `metrics.json` と `run_id` を見れば結果を追える

---

## ディレクトリの見方

| パス | 役割 |
|---|---|
| `ml/data/` | 前処理・特徴量生成 |
| `ml/training/` | 学習 |
| `ml/evaluation/` | 評価 |
| `ml/registry/` | 成果物保存 |
| `pipeline/` | data / training / evaluation の入口 |

---

## この Phase で押さえること

1. ML は `データ → 前処理 → 学習 → 評価 → 保存` の流れで理解する
2. `LightGBM` は学習器、`scikit-learn` は周辺用途という役割分担で見る
3. `metrics.json` と `run_id` を見れば学習結果を追える
