# 01_仕様と設計（Phase 1: ML Foundations）

## 位置付け

Phase 1 は **ML 基礎の学習専用フェーズ**。  
対象は California Housing の回帰問題で、範囲は以下に限定する。

- データ投入（seed）
- 前処理 / 特徴量生成
- 学習 / 評価
- アーティファクト保存 / 実験ログ

推論 API・UI・Port/Adapter・DI は **Phase 2 (`2/study-ml-app-pipeline/`)** の責務。

## アーキテクチャ

```text
PostgreSQL
  -> pipeline/data_job
  -> pipeline/training_job
      -> ml/data/preprocess
      -> ml/data/feature_engineering
      -> ml/training
      -> ml/evaluation
      -> ml/registry/artifacts/{run_id}
  -> pipeline/evaluation_job
```

## モジュール責務

- `ml/common/`
  - Phase 1 内で閉じる共通基盤（設定・ロギング・schema・run_id）
  - Phase 2 とは import 共有せず、必要コードは複製して独立運用
- `ml/data/`
  - `loaders`: PostgreSQL 取得
  - `preprocess`: 欠損補完 / 外れ値キャップ / 対数変換
  - `feature_engineering`: 派生特徴量追加
- `ml/training/`
  - LightGBM 学習
- `ml/evaluation/`
  - RMSE / R2 算出、metrics.json 保存
- `ml/registry/`
  - 学習成果物の保存先管理（`latest` シンボリックリンク運用）
- `pipeline/`
  - `data_job`, `training_job`, `evaluation_job` の 3 ジョブを保持
  - 推論系ジョブは Phase 2 に移管済み

## データフロー

1. `pipeline.data_job` が sklearn California Housing を PostgreSQL に投入
2. `pipeline.training_job` が train/test を取得して前処理・特徴量生成
3. LightGBM 学習後、RMSE/R2 を算出
4. `ml/registry/artifacts/{run_id}/` に `model.lgb` / `metrics.json` を保存
5. `latest` シンボリックリンクを更新

## 設定

`ml/common/config/base.py`（`BaseAppSettings`）を利用し、次の優先順位で読み込む。

1. 環境変数
2. `env/secret/credential.yaml`
3. `env/config/setting.yaml`
4. コード上の既定値

主要キー:

- `postgres_host`, `postgres_port`, `postgres_db`, `postgres_user`, `postgres_password`
- `model_dir`

## 運用上の非負制約

- Phase 1 は学習基礎フェーズとして維持し、API を再導入しない
- 学習タスクは LightGBM 回帰 + RMSE/R2 を維持
- Phase 2 とコード共有 import は行わず、フェーズ独立を優先
